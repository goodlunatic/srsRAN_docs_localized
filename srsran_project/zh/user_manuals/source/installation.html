安装指南

下载并构建 srsRAN Project 需要执行以下步骤：

  1. 安装依赖项

  2. 安装 RF 驱动程序（仅适用于 Split 8 部署）

  3. 克隆存储库

  4. 构建代码库

注意事项

srsRAN Project 需要基于 Linux 的操作系统，我们推荐 Ubuntu（22.04 或更高版本）。

构建工具和依赖项

srsRAN Project 使用 CMake 和 C++17。我们推荐以下构建工具：

  • cmake

  • 海湾合作委员会 （v11.4.0 或更高版本） 或 铿锵 （v14.0.0 或更高版本）

srsRAN Project 具有以下必要的依赖项：

  • libfftw

  • 库文件

  • yaml-cpp

  • mbedTLS

  • 谷歌测试

您可以安装各种发行版所需的构建工具和依赖项，如下所示：

sudo apt-get install cmake make gcc g++ pkg-config libfftw3-dev libmbedtls-dev libsctp-dev libyaml-cpp-dev libgtest-dev

sudo yum install cmake make gcc gcc-c++ fftw-devel lksctp-tools-devel yaml-cpp-devel mbedtls-devel gtest-devel

sudo pacman -S cmake make base-devel fftw mbedtls yaml-cpp lksctp-tools gtest

还建议用户安装以下内容（尽管不是必需的）：

  • ccache：有助于加快重新编译速度

  • backward-cpp：如果运行时发生错误，该库有助于在标准输出中生成更多信息的回溯

RF 驱动程序

注意事项

只有 Split 8 部署需要 UHD 和/或 ZMQ；如果您计划使用 Split 7.2 部署，可以跳过这一步。

srsRAN Project 使用 RF 驱动程序来支持不同的无线电类型。目前仅支持 UHD 和 ZMQ：

  • UHD （我们推荐 UHD 的 LTS 版本，即 3.15 或 4.0。）

  • ZMQ

克隆和构建

srsRAN Project 可以在启用或禁用某些功能的情况下构建。这是在构建过程中通过使用 CMake 标志和/或在构建代码之前下载第三方依赖项来完成的。以下各节概述了这些不同的构建选项。

首先，克隆 srsRAN Project 存储库：

git clone https://github.com/srsRAN/srsRAN_Project.git

然后构建代码库：

cd srsRAN_Project
mkdir build
cd build
cmake ../
make -j $(nproc)
make test -j $(nproc)

您现在可以从以下位置运行 gNB srsRAN_Project/build/apps/gnb/。如果您想安装srsRAN Project，可以使用以下命令：

sudo make install

注意事项

srsRAN Project 允许编译时选择 Split 7.2 或 Split 8 配置。默认情况下，srsRAN Project 构建时启用了这两个选项。如果您想使用两个可用的拆分配置选项进行编译，请遵循“Vanilla”安装指南。

首先，克隆 srsRAN Project 存储库：

git clone https://github.com/srsRAN/srsRAN_Project.git

然后构建代码库，确保传递正确的 CMake 标志：

cd srsRAN_Project
mkdir build
cd build
cmake -DDU_SPLIT_TYPE=SPLIT_7_2  ../
make -j $(nproc)
make test -j $(nproc)

您现在可以从以下位置运行 gNB srsRAN_Project/build/apps/gnb/。如果您想安装srsRAN Project，可以使用以下命令：

sudo make install

注意事项

srsRAN Project 允许编译时选择 Split 7.2 或 Split 8 配置。默认情况下，srsRAN Project 构建时启用了这两个选项。如果您想使用两个可用的拆分配置选项进行编译，请遵循“Vanilla”安装指南。

首先，克隆 srsRAN Project 存储库：

git clone https://github.com/srsRAN/srsRAN_Project.git

然后构建代码库，确保传递正确的 CMake 标志：

cd srsRAN_Project
mkdir build
cd build
cmake -DDU_SPLIT_TYPE=SPLIT_8  ../
make -j $(nproc)
make test -j $(nproc)

您现在可以从以下位置运行 gNB srsRAN_Project/build/apps/gnb/。如果您想安装srsRAN Project，可以使用以下命令：

sudo make install

在 Ubuntu 上，ZeroMQ 开发库可以通过以下方式安装：

sudo apt-get install libzmq3-dev

另外，ZeroMQ 也可以从源代码构建。

首先，需要安装 libzmq：

git clone https://github.com/zeromq/libzmq.git
cd libzmq
./autogen.sh
./configure
make
sudo make install
sudo ldconfig

二、安装czmq：

git clone https://github.com/zeromq/czmq.git
cd czmq
./autogen.sh
./configure
make
sudo make install
sudo ldconfig

最后，您需要编译srsRAN Project（假设您已经安装了所有必需的依赖项）。

注意事项

如果您在安装 ZMQ 和其他依赖项之前已经构建并安装了 srsRAN Project，则必须重新构建以确保正确识别 ZMQ 驱动程序。

以下命令可用于从源代码克隆和构建：

git clone https://github.com/srsran/srsRAN_Project.git
cd srsRAN_Project
mkdir build
cd build
cmake ../ -DENABLE_EXPORT=ON -DENABLE_ZEROMQ=ON
make -j`nproc`

警告

ZeroMQ 默认处于禁用状态，需要在运行 cmake 时加入 -DENABLE_EXPORT=ON -DENABLE_ZEROMQ=ON 才会启用。

特别注意 cmake 控制台输出。请务必阅读以下行以确保 srsRAN 已正确检测到 ZMQ：

...
-- FINDING ZEROMQ.
-- Checking for module 'ZeroMQ'
--   No package 'ZeroMQ' found
-- Found libZEROMQ: /usr/local/include, /usr/local/lib/libzmq.so
...

的 运行srsRAN 文档的部分进一步讨论如何配置和运行 gNB 应用程序。

软件包

srsRAN Project 可直接从各种 Linux 发行版的软件包中下载。寻求简单安装且不想编辑源代码的用户应使用软件包安装。

Ubuntu 用户可以使用以下命令下载 srsRAN Project 软件包：

sudo add-apt-repository ppa:softwareradiosystems/srsran-project
sudo apt-get update
sudo apt-get install srsran-project -y

Arch Linux 用户可以使用 AUR 帮助程序下载 srsRAN Project 软件包，例如‘yay’，使用以下命令：

yay -Sy srsran-project-git

这将从 git 安装最新版本的 srsRAN Project。

从软件包安装时，srsRAN Project 的示例配置可以在 /usr/share/srsran。有关这些配置文件的信息，请参阅 这里

然后可以使用以下命令运行该应用程序：

sudo gnb -c <config file>

PHY 测试向量

许多 PHY 测试基于 MATLAB 生成的测试向量。默认情况下，这些测试被禁用。 需要执行以下步骤来启用它们：

  1. 下载最新的 PHY 测试向量集.

  2. 将 PHY 测试向量提取到 srsRAN 工作目录中的位置：

tar -xf phy_testvectors.tar -C /path_to_your_local_repository/srsRAN_Project

  1. 通过重新生成 CMake 构建系统来启用 PHY 测试向量：

cmake -B build -DUSE_PHY_TESTVECTORS=ON

  1. 重建 srsRAN Project。
