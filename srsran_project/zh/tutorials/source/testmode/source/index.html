srsRAN gNB(-DU) 负载测试

简介

在本教程中，我们将指导您通过配置文件中的可用参数完成 srsRAN Project gNB(-DU) 应用程序的负载测试过程，无需特定硬件。

的 ru_dummy gNB 配置中的选项允许用户模拟 RU，并通过排除 OFH 层专门专注于测试 DU。 RU 模拟器 (ru_emu）可用于模拟流量 由拆分 7.2 配置中的 O-RU 发送和接收，同时还允许您监控关键性能指标 (KPI)。

对于没有核心网的场景， no_core gNB 配置中的标志允许 gNB(-DU) 在没有物理核心网络的情况下运行。

的 testmode gNB 应用程序中的功能旨在评估 srsRAN gNB(-DU) 在特定硬件上的性能。这个 功能模拟单个或多个 UE，模拟跨层（例如 MAC、PHY 和 OFH）的流量。通过将流量从 CU 和 RU 复制到 DU， testmode 提供了评估和优化系统性能的有效方法。直接集成到 gNB 应用程序中， testmode 不需要额外的 超越 gNB。

以下各节提供了这些场景的详细说明。有关这些模式的具体配置参数和使用的详细信息，请参阅 srsRAN Project 配置参考.

使用 ru_dummy

如果您想在没有物理或模拟 RU 的情况下进行负载测试，您可以使用 ru_dummy gNB 配置中的配置选项。这将从测试中排除 OFH 层，因为没有数据 通过网络发送。要使用 ru_dummy 参数将以下部分添加到您的 gNB 配置文件中：

ru_dummy:
  dl_processing_delay: 1
  time_scaling: 1

使用 ru_emu

RU 仿真器是一个额外工具，需要与 gNB 应用程序分开构建。要构建 RU 模拟器，请在 srsRAN Project 仓库中执行以下命令：

mkdir build && cd build
cmake ..
cd apps/examples/ofh
make -j $(nproc)

如果您想使用 DPDK 运行 RU 模拟器，请改用以下 cmake 命令：

cmake -ENABLE_DPDK=ON ..

注意事项

使用 DPDK 构建并不需要 ru_emu

创建一个名为的新配置文件 emu.yaml 包含以下内容：

log:
  filename: /tmp/ru_emu.log
  level: warning

ru_emu:
    cells:
    - bandwidth: 100                          # Bandwidth of the cell
      network_interface: ens2f1               # Use BDF instead of interface name for DPDK
      ru_mac_addr: 00:33:22:33:00:11          # MAC address of the RU
      du_mac_addr: 00:33:22:33:00:66          # MAC address of the DU
      enable_promiscuous: false               # Promiscuous mode flag
      vlan_tag: 33                            # VLAN tag
      dl_port_id: [0, 1, 2, 3]                # Port IDs for downlink
      ul_port_id: [0, 1, 2, 3]                # Port IDs for uplink
      prach_port_id: [4, 5]                   # Port IDs for PRACH
      compr_method_ul: "bfp"                  # Compression method for uplink
      compr_bitwidth_ul: 9                    # Compression bitwidth for uplink
      t2a_max_cp_dl: 470                      # T2a maximum value for downlink Control-Plane
      t2a_min_cp_dl: 350                      # T2a minimum value for downlink Control-Plane
      t2a_max_cp_ul: 200                      # T2a maximum value for uplink Control-Plane
      t2a_min_cp_ul: 90                       # T2a minimum value for uplink Control-Plane
      t2a_max_up: 345                         # T2a maximum value for User-Plane
      t2a_min_up: 70                          # T2a minimum value for User-Plane

# dpdk:
#   eal_args: "--lcores (0-1)@(0-15)"

调整上述参数以匹配您的配置。如果要使用DPDK，请在 network_interface 场。可以找到BDF 使用 dpdk-devbind.py -s 命令。另外，取消注释 dpdk 部分并提供正确的 eal_args 论据。

使用以下命令启动 RU 模拟器：

sudo ./ru_emulator -c emu.yaml

使用 no_core

如果您想在没有 5G 核心可用的情况下模拟核心网络，您可以使用 no_core gNB 配置中的标志。要使用 no_core flag 将以下部分添加到您的 gNB 配置中：

cu_cp:
  amf:
    no_core: false

使用 testmode

一旦 RU 和核心网络就位，您就可以开始使用 testmode。还可以在以下位置找到示例配置文件 srsRAN_项目/配置 之内 srsRAN Project源文件：

test_mode:
  test_ue:
    rnti: 0x44
    ri: 1 # Set to 2 or 4 for 2 layer or 4 layer MIMO operation
    cqi: 15
    nof_ues: 1
    pusch_active: true
    pdsch_active: true

此配置将模拟单个 UE RNTI = 0x44, CQI 设置为 15 并且 RI 至 1。

运行gNB(-DU)时，配置文件可以串联，这意味着用户可以轻松测试各种配置，而无需修改其基本配置。对于这个例子，上面的配置 将与示例配置连接起来 gnb_ru_ran550_tdd_n78_100mhz_4x2.yml 可以在以下位置找到： srsRAN_Project/configs。这将允许在不连接物理 UE 的情况下测试 RU。 这种能力扩展到其他前端，例如 USRP 和 ZMQ。

要运行所描述的场景，可以使用以下命令：

sudo ./apps/gnb/gnb -c gnb_ru_ran550_tdd_n78_100mhz_4x2.yml -c testmode.yml
