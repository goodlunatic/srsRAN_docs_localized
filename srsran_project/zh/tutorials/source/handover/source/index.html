srsRAN gNB 移交

注意事项

此用例需要 srsRAN Project 24.04 或更高版本。

概述

srsRAN Project 24.04 为代码库带来了切换功能。这些功能可实现 gNB 内切换，即连接到同一 CU-CP 的切换单元。这允许 UE 在 两个小区，可以通过命令行手动触发切换，也可以通过物理移动 UE 自动触发。

实际上，这是通过在 srsRAN gNB 二进制文件中的 DU 内创建两个单元来实现的。这需要使用具有两个独立 RF 链的 USRP。每个单元格都会被赋予一个关联的 RF 链条，UE 可以在这些链条之间移动。

下图展示了设置架构：

硬件和软件概述

本应用笔记使用以下硬件和软件：

  • 装有 Ubuntu 22.04.1 LTS 的电脑

  • srsRAN Project (24.04 或之后)

  • 埃特斯研究 X310 USRP （连接超过 10 GigE）

  • Open5GS 5G 核心 （在泊坞窗中运行）

  • COTS UE (Fairphone 5)

srsRAN Project

如果您尚未这样做，请安装最新版本的 srsRAN Project 及其所有依赖项。这在 安装指南.

X310 USRP

切换要求使用的 USRP 具有带有独立 RF 链的双通道 RF 前端。因此，本教程使用 X310 作为 RF 前端。

并非所有双通道 USRP 都有独立的 RF 链，这意味着 B200 系列 USRP 等不适合此用例。

Open5GS

出于本应用说明的目的，我们将使用 srsRAN Project 中提供的 Docker 化 Open5GS 版本，网址为 srsgnb/docker 作为 5G 核心。

Open5GS 是 5G 核心和 EPC 的 C 语言开源实现。以下链接将为您提供 包含下载和设置 Open5GS 所需的信息，以便可以与 srsRAN 一起使用：

  • GitHub

  • 快速入门指南

现货 UE

本教程使用支持 5G SA 的 COTS UE，特别是 费尔电话5。可以找到已使用 srsRAN Project 测试的 COTS UE 的详细列表 这里.

有关将 COTS UE 连接到 srsRAN Project 的更多信息，请参阅 完整教程.

配置

要配置 srsRAN Project 以启用小区之间的切换，必须执行以下步骤：

  • 将两个单元格添加到 细胞 清单

  • 更新 铜_cp 流动性 选项

可以在此处下载示例配置文件： 移交_x310.yml

配置更改的详细信息如下。

一个 细胞 创建的列表包含两个单元格。每个小区都被赋予不同的物理小区ID和PRACH根序列索引。

细胞:
-
  # 单元 1
  PCI: 1                              # 设置物理小区ID
  普拉赫:
    prach_root_sequence_index: 0        # 设置PRACH根序列索引
-
  # 单元 2
  PCI: 2                              # 设置物理小区ID
  普拉赫:
    prach_root_sequence_index: 64       # 设置PRACH根序列索引

小区和移动性信息也必须添加到 铜_cp，在 流动性 选项。首先是 细胞 列表需要更新每个单元格， 报告配置 也必须更新 具有每个小区的必要信息以及触发切换的条件。和往常一样， AMF 也配置在 铜_cp.

铜_cp:
  AMF:
    地址: 10.12.1.105                                           # AMF 的地址或主机名。
    绑定地址: 10.12.1.148                                      # gNB 为来自 AMF 的流量绑定的本地 IP。
    支持的跟踪区域:                                   # 配置CU-CP关联的TA
      - 塔克: 7                                                    # 追踪区号
        plmn_列表:                                                # 与跟踪区域关联的 PLMN 列表
          - 普鲁姆: "00101"                                         # PLMN ID
            tai_slice_support_list:                               # 跟踪区域支持的切片列表
              - 海温: 1                                              # 切片/服务类型
  流动性:
    从测量触发切换: 真实                    # 设置 CU-CP 在相邻小区测量到达时触发切换
    细胞:                                                      # cu-cp 已知的可用于切换的小区列表
      - 小区编号: 0x19B0                                        # 小区 1 的小区 ID
        周期报告cfg_id: 1                                 # 设置该小区的周期报告配置ID
        细胞:                                                   # 可用于切换的相邻小区
          - 小区编号: 0x19B1                                      # 可用于切换的邻区小区ID
            报告配置: [ 2 ]                                   # 报告要为此相邻小区配置的配置
      - 小区编号: 0x19B1                                        # 小区 2 的小区 ID
        周期报告cfg_id: 1                                 # 设置该小区的周期报告配置ID
        细胞:                                                   # 可用于切换的相邻小区
          - 小区编号: 0x19B0                                      # 可用于切换的邻区小区ID
            报告配置: [ 2 ]                                   # 报告要为此相邻小区配置的配置
    报告配置:                                             # 设置触发切换的报告配置
      - 报告_cfg_id: 1                                          # 报告配置 ID 1
        报告类型: 期刊                                   # 设置报告类型为定期
        报告间隔毫秒: 480                                   # 设置每 480ms 报告一次
      - 报告_cfg_id: 2                                          # 报告配置 ID 2
        报告类型: 事件触发                              # 设置报告类型为事件触发
        事件触发报告类型: a3                           # 设置事件触发报告类型为A3
        测量触发数量: 参考值                               # 设置测量触发数量为rsrp
        meas_trigger_quantity_offset_db: 3                        # 设置测量触发量偏移为3dB
        迟滞_db: 0                                          # 将迟滞设置为 0dB
        触发时间毫秒: 100                                   # 设置触发时间为100ms

A3事件定义为应触发同频切换的事件。在上面的配置中，发生此类事件的条件 当相邻小区的 RSRP 测量值比服务小区好 1.5 dB 时，应设置为触发。迟滞设置为0，这意味着切换 当恰好满足 1.5 dB 偏移时触发。一旦满足这些条件，UE 将在小区之间切换。对于每个方向的切换都是如此。

连接 COTS UE

将 COTS UE 连接到网络遵循中概述的相同步骤 本教程。有关与此相关的调试技巧，请参阅教程 GitHub 讨论.

触发切换

切换可以通过两种方式触发：

  • 在 gNB 运行时从控制台调整 Tx 增益

  • 将 UE 从左向右物理移动，远离主电池并向辅助电池移动。

从控制台

每个单元的 Tx 增益可以在运行时从控制台手动控制。

当gNB运行时，您可以使用以下命令动态调整Tx增益：

tx_gain <port_id> <gain_dB>

注意事项

Rx 增益也可以以相同的方式设置，使用 接收增益 <端口号> <增益_dB>。要触发切换，只需修改 Tx 增益。

在此示例中，具有 PCI 1 的单元的增益将发生变化（这对应于 USRP 上 RF0 的输出），而具有 PCI 2 的单元的增益将被固定（这对应于 USRP 上 RF1 的输出）。将增益降低至 10 dB，会导致 如果 UE 移至 PCI 2，则将增益增加回 30 dB 将导致 UE 移回 PCI 1 单元。

可以使用以下命令调整增益：

tx_gain 0 10

身体上

下图显示 UE 沿 Y 轴上下移动，这会导致 UE 在小区之间切换，因为随着 UE 远离服务小区，服务小区的信号强度会减弱。相反的情况发生在相邻小区上， 当 UE 进入其服务区域时，信号强度会增加。一旦满足配置文件中定义的条件，这将触发 A3 事件并导致 UE 在小区之间切换。在这种情况下，一旦 RSRP 相邻小区相对于当前服务小区有 1.5 dB 的偏移。

从物理上讲，这意味着在 USRP 的天线之间从左到右移动 UE，其中 PCI 1 对应于与 RF0 关联的单元，PCI 2 对应于与 RF1 关联的单元。
