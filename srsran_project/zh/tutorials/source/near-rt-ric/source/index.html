O-RAN NearRT-RIC 和 xApp

概述

本应用笔记展示了如何使用 srsRAN Project gNodeB 公开的 E2 接口，并展示其与第三方 RIC 框架的互操作性。 为此，我们使用 O-RAN联盟 中提供的兼容的 NearRT-RIC 和 xApp 奥兰SCRIC 和 柔性RIC 项目。

我们的 E2 接口实现基于以下 O-RAN 技术规范：

  • O-RAN.WG3.E2AP-R003-v03.00

  • O-RAN.WG3.E2SM-R003-v03.00

  • O-RAN.WG3.E2SM-KPM-R003-v03.00

  • O-RAN.WG3.E2SM-RC-R003-v03.00

设置概述

硬件和软件概述

本应用笔记使用以下硬件和软件：

  • 装有 Ubuntu 22.04.1 LTS 的电脑

  • srsRAN Project

  • srsRAN UE （srsRAN 4G 23.04 或更高版本）

  • 零MQ

  • 奥兰SCRIC 或 柔性RIC

  • Open5GS 5G 核心

  • Wireshark（版本4.0.7或更高版本）

局限性

虽然我们的最终目标是完全支持 E2 接口，但它仍在开发中，当前版本的功能和操作受到限制。 具体来说，当前的E2接口实现仅支持E2SM_KPM和E2SM_RC服务模型，具有以下限制：

  • E2SM_RC服务模型：

    • 仅支持控制服务样式 2

  • E2SM_KPM服务模型：

    • 支持所有报表服务样式 (1 - 5)

    • 监控周期限制为1s

    • 以下 3 个“虚拟”DU 指标已公开（它们将在未来版本中删除）：

      • 消费者质量指数

      • RSRP

      • RSRQ

    • 公开了以下 6 个 ORAN 定义的指标：

      • DRB.UEThpDl - 深度学习吞吐量

      • DRB.UEThul - 上行吞吐量

      • DRB.Rlc数据包丢弃率Dl - UL数据包成功率

      • DRB.数据包成功率UlgNBUu - RLC DL丢包率

      • DRB.RlcSdu传输量DL - RLC DL 传输的 SDU 音量

      • DRB.RlcSdu 传输量 UL - RLC UL 传输的 SDU 音量

安装

为了展示 srsRAN Project gNodeB 公开的 E2 接口的用法，我们使用 NearRT-RIC 以及来自 ORAN SC RIC 和 FlexRIC 框架的示例 KPM 监控 xApp。

注意事项

根据您使用的 RIC，您应该从下面的选项卡中选择适当的说明。整个教程中的选项卡都是分组的，因此您只需在本节中选择适当的说明即可。

O-RAN 软件社区 (SC) 近实时 RIC 是一个参考平台，符合 O-RAN 联盟工作组创建的架构和规范。

ORAN SC RIC平台是一个先进的软件框架，其功能包括平台健康监控、警报等。 由于它依赖于 Kubernetes 和 Helm 来实现部署、可靠性和可扩展性，因此其普通安装相当复杂，涉及许多步骤，并且需要对这些框架有高水平的知识和专业知识。

在本应用笔记中，我们使用 O-RAN SC 近 RT RIC 的最小版本（我发布），可以使用单个 Docker 命令轻松部署为多容器应用程序，从而无需使用 Kubernetes 或 Helm。 具体来说，我们将使用 Docker 和本文档中提供的配置文件来部署 ORAN SC RIC 存储库.

ORAN SC RIC部署执行如下：

git clone https://github.com/srsran/oran-sc-ric
光盘 ./oran-sc-ric
docker compose up

的 柔性RIC 框架提供 O-RAN联盟 兼容的 E2 节点代理模拟器、NearRT-RIC 和用 C/C++ 和 Python 编写的 xApp。

O-RAN 规范正在不断发展，FlexRIC 正在开发多个轨道（即 git 分支）。 在本教程中，我们使用 br-flexric 分支（提交： 1a3903a7).

FlexRIC 需要安装以下依赖项（请参阅 FlexRIC 安装指南 详情）：

sudo apt-get update
sudo apt-get install swig libsctp-dev python3 cmake-curses-gui python3-dev pkg-config libconfig-dev libconfig++-dev

FlexRIC安装执行如下：

git clone https://gitlab.eurecom.fr/mosaic5g/flexric.git
光盘 flexric
git checkout br-flexric
mkdir build
光盘 build
cmake -DKPM_VERSION=KPM_V3_00 -DXAPP_DB=NONE_XAPP ../
make
sudo make install

请注意，虽然默认情况下 Ubuntu 22.04.1 使用 海湾合作委员会-11，所使用的 FlexRIC 版本只能使用 海湾合作委员会-10。一种可能的切换方式 海湾合作委员会 版本是使用 更新替代方案 工具，例如：

sudo update-alternatives --config gcc

There are 3 choices 为了 the alternative gcc (providing /usr/bin/gcc).

* 1            /usr/bin/gcc-10   10        manual mode
  2            /usr/bin/gcc-11   11        manual mode
  3            /usr/bin/gcc-9    9         manual mode

零MQ

在 Ubuntu 上，ZeroMQ 开发库可以通过以下方式安装：

sudo apt-get install libzmq3-dev

另外，ZeroMQ 也可以从源代码构建。

首先，需要安装 libzmq：

gNB

这里，我们描述与E2代理相关的gNB配置参数。

在所有DU中启用E2代理并启用E2SM_KPM服务模块：

  启用_du_e2: 真实                # 启用 DU E2 代理（每个 DU 实例一个）
  e2sm_kpm_启用: 真实            # 启用KPM服务模块
  e2sm_rc_启用: 真实             # 启用RC服务模块
  地址: 127.0.0.1                   #RIC IP 地址
  港口: 36421                       # RIC端口

启用 E2AP 数据包捕获并设置输出 pcap 文件的名称：

聚氯乙烯:
  e2ap_启用: 真实                              # 设置为 true 以启用 E2AP PCAP。
  e2ap_du_文件名: /tmp/gnb_du_e2ap.pcap        # DU E2AP PCAP 的存储路径。
  e2ap_cu_cp_文件名: /tmp/gnb_cu_cp_e2ap.pcap  # CU-CP E2AP PCAP 的存储路径。
  e2ap_cu_up_文件名: /tmp/gnb_cu_up_e2ap.pcap  # CU-UP E2AP PCAP 的存储路径。

启用 启用 RLC 和 DU 调度程序指标报告，该报告将为 E2SM_KPM 服务模型提供测量数据：

指标:
  层数:
    启用RLC: 真实                # 启用 RLC 指标
    启用_计划: 真实              # 启用 DU 调度程序指标
  周期性:
    du_report_period: 1000          # 设置DU统计报告周期为1s

IP 流量与 ping

Ping是测试网络端到端连通性最简单的工具，即测试UE和核心是否可以通信。在这里，我们使用它从 UE 生成流量，因此 gNB 可以测量与数据传输相关的指标（例如吞吐量）。

要从 UE 运行 ping 到核心，请使用：

sudo ip netns 执行 ue1 ping -i 0.1 10.45.1.1

注意，我们将 ping 间隔设置为 0.1 秒，以增加流量。

示例 平 输出：

E2AP数据包分析仪

启用 E2AP PCAP

您可以通过以下方式启用 E2AP PCAP 本指南.

实时捕捉

Wireshark可用于收集运行时E2代理（位于srsRAN gNB）和NearRT-RIC之间交换的E2AP数据包。这需要执行以下步骤：

  1. 开始在环回接口上嗅探。

  2. 将过滤器设置为 sctp.端口 == 36421.

  3. 右键单击任何数据包 -> 解码为.. -> 将当前设置为 E2AP

  4. 现在过滤器可以设置为 电子2AP 仅显示 E2AP 消息。

请注意，至少需要 Wireshark 版本 4.0.7 才能正确解码和显示 E2AP 数据包（即，早期的 Wireshark 版本不支持 E2APv3 协议，因此将显示有关格式错误的数据包的信息）。

UE问题

如果UE无法连接网络，请确保正确 单元格配置 参数在gNB中设置。

如果 UE 正在连接，但没有建立 PDU 会话，您应该检查以下内容：

  • UE 和 Core 的 APN 配置相同

  • 您正在使用最新版本的 srsUE

  • 核心的 IP 转发已启用，您可以通过以下方式执行此操作 本指南.

  • UE 的 IP 转发已启用，请参阅以下部分

UE IP转发

为了确保 UE 流量正确发送到互联网，必须启用正确的 IP 转发。应启用 IP 转发 主机，即运行 Open5GS docker 容器的容器。 这可以通过以下命令来完成：

sudo sysctl -w net.ipv4.ip_forward=1
sudo iptables -t nat -A POSTROUTING -o <IFNAME> -j MASQUERADE

哪里 <IFNAME> 是连接到互联网的接口的名称。

要检查配置是否正确，请运行以下命令：

sudo ip netns 执行 ue1 ping -i 1 8.8.8.8

如果UE能够ping通Google DNS，则可以成功访问互联网。

第二个 Open5GS 实例（手动安装）

主机 PC 上的 IP 路由条目： 10.45.0.0 和 10.53.1.0 应该使用相同的接口，例如：

route -n

Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
0.0.0.0         192.168.0.1     0.0.0.0         UG    100    0        0 eno1
10.45.0.0       10.53.1.2       255.255.0.0     UG    0      0        0 br-dfa5521eb807
10.53.1.0       0.0.0.0         255.255.255.0   U     0      0        0 br-dfa5521eb807
...

但是，如果 Open5GS 的第二个实例（手动安装）正在主机 PC 上运行，则路由到 10.45.0.0 去 奥格斯顿 界面。因此，UE 无法访问互联网，因为主机将向手动安装的 Open5GS 版本发送数据包。 要解决此路由问题，您可以禁用（甚至删除）手动安装的 Open5GS – 请检查第 6 和/或 7 节 Open5GS教程。 此外，您可能需要禁用 奥格斯顿 使用以下命令进行界面：

sudo ifconfig ogstun 0.0.0.0 down

RIC在不同的机器上运行

如果您在另一台机器上运行 RIC，则需要正确配置 E2绑定地址 gNB 配置文件中的参数。这显示在示例配置中，并注释掉了该行。如果您在单独的计算机上运行 RIC，只需取消注释此选项即可。
