Kubernetes 上的 srsRAN gNB

简介

本教程概述了使用以下命令为拆分 7.2 架构部署 srsRAN gNB 所需的步骤 Kubernetes。这种方式非常适合需要网络的场景 长期部署和管理，具有高服务可用性和增强的容错能力。

简而言之，Kubernetes可以描述如下：

在管理测试网络时，Kubernetes 在简化操作方面发挥着关键作用。 它提供了一个统一的平台来编排网络功能、优化资源使用、 并根据需求自动扩展。其内置的容错能力确保服务的一致性 可用性，同时其对持续部署的支持加速了创新。 Kubernetes 还简化了跨不同环境的部署，提高了适应性。 本质上，它为高效的网络功能管理提供了一个内聚且可扩展的框架。

但请注意，此类部署对于以研究和开发为重点的用例来说可能并不理想 需要微调配置文件或修改源代码。用于迭代开发 和测试时，“裸机”设置通常更合适。

CU/DU

CU/DU 由 srsRAN Project gNB 提供。开放前传 (OFH) 库提供 DU 和 RU 之间所需的接口。

鲁

对于本教程，您可以使用 srsRAN 支持的任何 RU。欲了解更多信息 支持的 O-RU，请参阅 本节 RU 教程。

5G核心

本教程使用 Open5GS 5G 内核。

Open5GS 是 5G 核心和 EPC 的开源实现，用 C 编写。 以下链接提供有关下载和设置 Open5GS 的指导 与 srsRAN 一起使用：

  • Open5GS GitHub

  • Open5GS 快速入门指南

时钟与同步

分离式 7.2 接口要求 DU 和 RU 之间紧密同步。 O-RAN WG4 定义了与开放前传一起使用的各种同步机制， 概述于 O-RAN.WG4.CUS.0-R003-v11.00，第 11 节。

本教程介绍如何配置 LLS-C1 和 LLS-C3 设置。

  • LLS-C1：DU作为PTP大师，RU作为PTP客户端。

  • LLS-C3：DU 和 RU 都是 PTP 客户端，同步到公共大师。

PTP 主时钟通常是 GPS 或铷时钟，而 PTP 客户端 通常是支持 PTP 的网络接口卡 (NIC)。

设置 K8s/K3s 裸机集群

1. 部署Kubernetes集群

Kubernetes 的安装因发行版和用于部署的工具而异。取决于 根据您的要求和环境，您可以选择最适合您需求的工具。在这个 教程中，我们在 Ubuntu 24.04 Server 上部署单节点 K3s 集群。 K3s是一款轻量级的Kubernetes 易于安装和管理的发行版。它专为资源受限的环境而设计 和边缘计算，使其成为裸机 Kubernetes 部署的绝佳选择。

用于部署 Kubernetes 的流行工具包括：

  • 库贝喷雾

  • 库贝德姆

  • K3s

  • 牧场主

K3s的安装非常简单，只需一条命令即可完成。以下命令安装 您服务器上的 K3：

curl -sfL https://get.k3s.io | sh -

欲了解更多信息，请参阅 K3s官方文档.

2.安装实时内核

Ubuntu 24.04 LTS 中的实时内核基于 PREEMPT_RT 补丁构建，可确保低延迟和确定性 时间敏感操作的性能。通过优先考虑关键流程并确保可预测的响应 有时，它是电信应用的理想选择。此版本还改进了对 Raspberry Pi 硬件的支持， 跨不同应用程序实现优化的实时计算。

要在 Ubuntu 24.04 上安装实时内核，您必须获得免费的 Canonical Pro 订阅。 注册于 规范网站 并创建一个帐户。之后，使用您的 Pro 令牌和以下命令来安装内核：

sudo pro attach <your-token>
sudo pro enable realtime-kernel

安装完成后重新启动系统。欲了解更多信息，请参阅 Ubuntu 文档.

3.安装TuneD

有关使用 TuneD 进行性能调整的信息，请参阅 srsRAN 性能调优指南 在我们的文档中。

4.安装DPDK

有关 DPDK 安装说明，请参阅 DPDK指南.

设置 PTP 同步

可以使用以下工具建立 PTP 同步 点对点4l, ts2phc, 和 PHC2系统。这些工具可以部署 使用 srsRAN Project linuxptp 舵图。第一步，安装 srsRAN Project Helm 存储库：

helm repo add srsran https://srsran.github.io/srsRAN_Project_helm/

根据您的设置，PTP 组件可以部署在不同的配置中。最常见的是 LLS-C1 和 LLS-C3，可以使用单播或多播传输。

  • 在 LLS-C1 配置中，DU服务器驱动PTP同步，RU作为客户端。 RU 接收来自 DU 的 PTP 消息。

  • 在 LLS-C3 配置中，DU 和 RU 都是从公共 PTP 主大师接收 PTP 消息的客户端。

在本教程中，我们演示如何使用 G.8275.1 多播配置文件部署 LLS-C1 和 LLS-C3 配置 的 linuxptp。欲了解更多信息，请参阅 linuxptp 官方文档.

设置 gNB

要部署 gNB，请编辑 值.yaml 文件并设置所需的 RAN 参数。一个例子 值.yaml 对于 srsRAN Project 可以找到 Helm Chart 这里.

如果您尚未添加 srsRAN Project Helm 存储库，请使用以下命令添加它：

helm repo add srsran https://srsran.github.io/srsRAN_Project_helm/

下面，我们将解释如何使用 srsRAN Helm Chart 设置不同的场景。

1. 连接集群内部核心网络

当所有组件在同一 Kubernetes 集群中运行时，您可以使用 DNS 主机名而不是 LoadBalancer。 例如，如果 Open5GS 核心网络部署在同一集群中，则使用 AMF 服务的主机名连接到它。

要确定集群域，请运行：

kubectl run -it --image=ubuntu --restart=Never shell -- sh -c 'apt-get update > /dev/null && apt-get install -y dnsutils > /dev/null && nslookup kubernetes.default | grep Name | sed "s/Name:\skubernetes.default//"'

输出示例：

If you dont see a command prompt, try pressing enter.
debconf: delaying package configuration, since apt-utils is not installed

.svc.kubernetes.local

在这种情况下，集群域是 svc.kubernetes.local。要构造服务主机名：

<service-name>.<namespace>.svc.<cluster-domain>

列出所有可用的服务：

kubectl get services -A

输出示例：

NAMESPACE     NAME               TYPE        CLUSTER-IP       EXTERNAL-IP   PORT(S)                  AGE
default       kubernetes         ClusterIP   10.96.0.1        <none>        443/TCP                  10d
default       open5gs-amf-ngap   ClusterIP   10.111.110.41    <none>        38412/SCTP               16h
[...]

这里，AMF服务名称是 open5gs-amf-ngap 命名空间是 默认。因此，主机名是：

open5gs-amf-ngap.default.svc.kubernetes.local

在中使用该主机名 AMF gNB 配置部分 值.yaml.

更多信息请参考官方 Kubernetes DNS 文档.

2. 在不使用 SR-IOV 插件的情况下分配 DPDK 设备

要在不使用 SR-IOV 插件的情况下将 PF 或 VF 直接分配给容器，您必须向 Pod 授予完全权限 访问主机系统。在 值.yaml，设置以下内容：

启用对主机网络的访问：

network:
    hostNetwork: true

启用特权模式并设置所需的功能：

securityContext:
    capabilities:
        add: ["SYS_NICE", "NET_ADMIN"]
    privileged: true

通过此设置，gNB Pod 可以完全访问主机网络堆栈。这使得 Pod 能够访问 外部和内部 Kubernetes 网络资源。

负载测试

下面，我们介绍两种测试系统最大负载的方法。

1.srsRAN RU模拟器

srsRAN RU 模拟器是模拟无线电单元 (RU) 的工具。它打印关键绩效指标，例如提前和延迟的数据包， 这对于调试网络问题和评估部署可以处理的负载很有用。你可以 使用专用 Helm 图表快速部署 RU 模拟器。

在部署 RU 模拟器之前，您必须获取 RU 和 DU MAC 地址，以及带宽、VLAN 标记、 和压缩方法。这些参数是 强制性的:

  • ru_mac_地址：用于开放前传 (OFH) 流量的接口的 MAC 地址。

  • 杜_mac_地址：用于 OFH 流量的 DU 接口的 MAC 地址。

配置示例：

ru_emu:
    cells:
    - bandwidth: 100
      network_interface: enp4s0f0
      ru_mac_addr: 50:7c:6f:45:44:33
      du_mac_addr: 00:11:22:33:44:00
      vlan_tag: 6
      ul_port_id: [0]
      compr_method_ul: "bfp"
      compr_bitwidth_ul: 9

在里面使用以下配置 值.yaml 文件以启用 RU 模拟器：

securityContext:
    capabilities:
        add: ["SYS_NICE", "NET_ADMIN"]
    privileged: true

另请确保通过设置显式禁用 SR-IOV：

sriovConfig:
    enabled: false

确保 网络接口 和 杜_mac_地址 值已针对您的部署正确设置。

2. 使用循环测试评估最大延迟

循环测试 是一种用于测量实时系统上的应用程序延迟的工具。评估最大延迟 在您的系统上，您可以部署 循环测试 使用 srsRAN Project 实时测试 舵图。

rt-test Helm Chart 旨在运行两个实时测试工具： 循环测试 和 压力NG。同时 循环测试 测量系统延迟和抖动， 压力NG 生成高 CPU 和内存负载来模拟系统压力。跑步 两者结合在一起可以让您评估系统在负载下保持实时性能的情况。

要配置您的测试场景，请更新 配置 的部分 值.yaml 文件。例如：

config:
    rt_tests.yml: |-
        stress-ng: "--matrix 0 -t 12h"
        cyclictest: "-m -p95 -d0 -a 1-15 -t 16 -h400 -D 12h"

此配置运行 压力NG 使用 12 小时 –矩阵 工作量和启动 循环测试 固定到 CPU核心1-15跨16个高优先级线程，运行12小时。欲了解更多信息 循环测试 和 压力NG，请参阅循环测试 和 压力NG 文档。

部署 实时测试，使用以下 Helm 命令：

helm install rt-tests srsran/rt-tests -n srsran --create-namespace

如果您尚未添加 srsRAN Helm 存储库，请立即添加：

helm repo add srsran https://srsran.github.io/srsRAN_Project_helm/

Grafana 仪表板带有预配置的 值.yaml 文件。唯一必须调整的字段是 簇域，这是解析服务主机名所必需的。

要确定您的集群域，请运行：

kubectl run -it --image=ubuntu --restart=Never shell -- sh -c 'apt-get update > /dev/null && apt-get install -y dnsutils > /dev/null && nslookup kubernetes.default | grep Name | sed "s/Name:\skubernetes.default//"'

wget https://raw.githubusercontent.com/srsran/srsRAN_Project_helm/refs/heads/main/charts/grafana-srsran/values.yaml
