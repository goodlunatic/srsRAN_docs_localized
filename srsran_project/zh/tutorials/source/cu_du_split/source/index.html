O-RAN CU-DU 分体式

注意事项

需要最新版本的 srsRAN Project 才能实现 CU-DU 与 srsRAN Project 的拆分。

概述

本教程概述了配置和运行 srsCU 和 srsDU 应用程序所需的步骤，以创建具有 CU-DU 拆分的 E2E O-RAN 兼容网络。在本教程中，USRP 用作 RF 前端，结果如下 是一个 分裂 8 配置。 要实施 Split 7.2x 配置，请结合使用本指南 RU指南.

硬件和软件概述

本应用笔记使用以下硬件和软件：

  • 装有 Ubuntu 22.04.1 LTS 的电脑

  • srsRAN Project （最新版本）

  • 埃特斯研究 B210 USRP （通过 USB 3.0 连接）

  • Open5GS 5G 核心 （裸机运行）

  • COTS UE (小米 12 5G)

srsRAN Project

如果您尚未这样做，请安装最新版本的 srsRAN Project 及其所有依赖项。这在 安装指南.

B210

本示例使用 USRP B210，它必须通过 USB 3.0 连接到 PC。外部时钟的使用不是强制性的，但对于连接不稳定或 UE 难以连接的设置，建议使用外部时钟。

Open5GS

对于本示例，Open5GS 在主机上运行裸机。

Open5GS 是 5G 核心和 EPC 的 C 语言开源实现。以下链接将为您提供 包含下载和设置 Open5GS 所需的信息，以便可以与 srsRAN 一起使用：

  • GitHub

  • 快速入门指南

现货 UE

本教程使用支持 5G SA 的 COTS UE，特别是 小米 12 5G。可以找到已使用 srsRAN Project 测试的 COTS UE 的详细列表 这里.

有关将 COTS UE 连接到 srsRAN Project 的更多信息，请参阅 完整教程.

配置

对于 CU-DU 拆分，需要两个配置文件，一个用于 srsCU，另一个用于 srsDU。这些配置文件都有详细解释 这里.

核心

如前所述，本示例中 Open5GS 运行裸机。无需更改配置，只需注册正在使用的 UE 的凭据（如果您尚未注册）即可。上面链接的快速入门指南概述了如何配置核心。

srsCU

要配置 srsCU， AMF, f1ap 和 f1u 字段必须在两个中正确配置 铜_cp 和 cu_up。以下配置文件显示了配置 srsCU 的最低要求：

铜_cp:
  AMF:
    地址: 127.0.1.100                     # AMF 的地址或主机名。
    绑定地址: 127.0.1.1                  # gNB 为来自 AMF 的流量绑定的本地 IP。
    支持的跟踪区域:             # 配置CU-CP关联的TA
      - 塔克: 7
        plmn_列表:
          - 普鲁姆: "00101"
            tai_slice_support_list:
              - 海温: 1
  f1ap:
    绑定地址: 127.0.10.1                 # 配置F1AP绑定地址，这将使CU-cp能够连接到DU

cu_up:
  f1u:
    插座:                               # 为 F1-U 接口定义 UDP/IP 套接字。
      -                                     # 套接字 1
        绑定地址: 127.0.10.1                  # 设置 F1-U 套接字将绑定到的地址。

的 AMF 参数特定于核心的本地配置。如果您通过 srsRAN Project 提供的 docker 脚本运行 Open5GS，您的配置将会有所不同。如果你有同样的情况 对 Open5GS 的配置方式进行了任何其他本地更改。

srsDU

要配置 srsDU， f1ap 和 f1u 必须配置参数，以及 俄罗斯特别提款权 和 单元格配置 参数。与 srsCU 一样，以下是配置 srsDU 的最低要求：

f1ap:
  cu_cp_地址: 127.0.10.1                    # CU-CP的地址
  绑定地址: 127.0.1.2             

f1u:
  插座:
    -
      绑定地址: 127.0.1.2

俄罗斯特别提款权:
  设备驱动程序: 超高清
  设备参数: 类型=b200，num_recv_frames=64，num_send_frames=64
  斯拉特: 23.04
  otw_格式: SC12
  发送增益: 80
  接收增益: 40

单元格配置:
  dl_arfcn: 650000
  乐队: 78
  信道带宽MHz: 20
  通用_scs: 30
  普鲁姆: "00101"
  塔克: 7
  PCI: 1

在此示例中，DU 配置为与 USRP B210 配合使用，并创建 20 MHz 单元。正在使用的 RU 和所需单元的细节可以根据需要进行更改。的 f1ap 配置必须与 CU 中的相关配置保持一致。

运行网络

必须遵循以下运行顺序才能正确初始化网络：

  1. Open5GS

  2. srsCU

  3. srsDU

核心

如果正确遵循 Open5GS 文档，则核心应该已经作为服务在后台运行。如果没有，则按照 Open5GS 文档中的步骤启动核心。

srsCU

首先，导航到 srsCU 应用程序文件夹。这可以通过以下命令来完成：

光盘 ~/srsRAN_Project/build/apps/cu

要运行 srsCU，可以使用以下命令（假设 srsCU 配置文件也位于同一文件夹中）：

sudo ./srscu -c cu.yml

如果 srsCU 正常运行，您应该在控制台中看到以下内容：

--== srsRAN CU (commit 2be82d8ea3) ==--

N2: Connection to AMF on 127.0.1.100:38412 completed
F1-C: Listening 为了 new connections on 127.0.10.1:38472...
==== CU 开始了 ===
Type <h> to view 帮助

srsDU

srsDU 的运行方式与 srsCU 相同。

首先，导航到正确的文件夹：

光盘 ~/srsRAN_Project/build/apps/du
