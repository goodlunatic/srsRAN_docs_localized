O-RAN 7.2 RU 指南

概述

srsRAN Project 支持与无线电单元 (RU) 的分离 7.2 和分离 8 前传接口。本教程概述了通过 split 7.2 将 RU 与 srsRAN CU/DU 连接所需的一般步骤。如需更详细的说明 在特定 O-RU 上选择一个 RU这个清单.

分裂 7.2 是 O-RAN 联盟发布的开放规范，旨在确保不同 DU 和 RU 解决方案之间的互操作性。

srsRAN 通过开放前传 (OFH) 库支持分离式 7.2 接口。 OFH 由 SRS 团队开发，是一个开源、可移植的库，具有最少的第三方依赖性。它的设计目的是最大限度地减少集成 以及与使用 srsRAN 和第三方 O-RU 相关的配置负担。

设置注意事项

本教程使用以下硬件：

  • 服务器（运行 srsRAN Project CU/DU）

    • CPU：AMD锐龙7 5700G

    • 内存：64GB

    • 网卡：Intel Corporation 82599ES 10 GB

    • 操作系统：Ubuntu 22.04（5.15.0-1037-实时）

  • 前传交换和同步

  • 鲁

  • COTS UE (OnePlus 9)

使用以下软件：

  • srsRAN Project

  • Open5GS 5G 核心

CU/DU

CU/DU 由 srsRAN Project 提供。开放前传 (OFH) 库提供 DU 和 RU 之间必要的接口。 DU 通过 SFP+ 光缆连接到前传交换机。

鲁

用户应选择最适合其用例和特定要求的 RU。有多种 O-RU 可供选择，具有适合室内和室外使用的特定实现方式、不同的价位和不同的硬件功能。已测试 O-RU 列表 相关硬件集成信息可参见 硬件集成，其中也包含如何将它们与 srsRAN Project 一起使用的详细说明。

对于所有设置，RU 应通过主前传接口通过 SFP+ 光缆连接到前传交换机。

5G核心

在本示例中，我们使用 Open5GS 5G 内核。

Open5GS 是 5G Core 和 EPC 的 C 语言开源实现。以下链接将为您提供 包含下载和设置 Open5GS 所需的信息，以便可以与 srsRAN 一起使用：

  • GitHub

  • 快速入门指南

同步

分离式 7.2 接口需要 DU 和 RU 之间严格的时序同步。 O-RAN WG 4 定义了与开放前传一起使用的各种同步方法。 O-RAN.WG4.CUS.0-R003-v11.00 第 11 节中概述了这些内容。

在此设置中，我们使用 LLS-C3。 LLS-C3 配置支持在中心站点和远程站点之间从 PRTC/T-GM 到 RU 分配网络定时。简单来说，它允许前传网络中的一个或多个 PRTC/T-GM 设备（充当 PTP 主设备）同步，以将网络定时信号传输到 DU 和 RU 组件，如上图所示。 在我们的设置中，前传交换机充当 PTP 主控（通过 GPS 同步），为 RU 和 DU 提供定时。它们通过以太网连接到交换机上的 SFP+ 10G 端口。

注意事项

OFH 库支持 O-RAN WG4 定义的所有已定义时钟模型和同步拓扑。 LLS-C3 的使用特定于此示例。

开关

所选交换机应该是具有定时感知功能的 O-RAN 交换机和 PTP 主交换机。这用于为 DU 和 RU 提供时钟和定时同步。

配置

开关

请参阅具体的 Switch 文档以正确配置它。特别是可能需要配置的任何计时和路由选项。

我们建议使用制造商文档以及 SRS 的特定交换机指南（如果在 集成指南.

CU/DU

网卡配置

DU 计算机应在 NIC 中启用巨型帧，并且应检查 PTP 进程以确保其正确同步。

要设置 NIC 中的巨型帧，请使用以下命令进行临时配置：

ifconfig <eth0> mtu 9600 up

哪里 eth0 是将 DU 连接到前传交换机的 SFP+ 光缆的以太网端口。

点对点配置

PTP 用于同步 DU 与前传交换机。 PTP 进程应在 DU 上运行，并且应检查 PTP 同步以确保其正确同步。 ptp4l 和 phc2sys 是与 linuxptp 进行 PTP 同步的两个主要组件。为了避免 PTP 出现任何问题，建议禁用 DU 上的网络时间协议 (NTP)。 使用以下命令在 Ubuntu 22.04 上禁用 NTP：

sudo timedatectl set-ntp false

我们已经为 LinuxPTP 版本 4 创建了 G.8275.1 组播 PTP 配置文件的示例配置文件。可以下载 here.

使用以下命令安装 LinuxPTP v4：

git clone http://git.code.sf.net/p/linuxptp/code linuxptp
cd linuxptp/
git checkout v4.2
make
sudo make install

使用以下命令运行 ptp4l：

sudo ptp4l -2 -i enp1s0f0 -f ./configs/default.cfg -m

然后您应该看到以下输出：

ptp4l[4321.966]: rms    6 max   14 freq -25784 +/-   9 delay   172 +/-   1
ptp4l[4323.091]: rms    5 max   10 freq -25778 +/-   8 delay   170 +/-   1
ptp4l[4324.216]: rms    6 max   11 freq -25781 +/-   9 delay   169 +/-   1
ptp4l[4325.341]: rms    5 max   10 freq -25783 +/-   8 delay   170 +/-   1

在上面的输出中，rms 值可用于确定 PTP 同步是否正确，为此我们寻找 < 10 的值。

接下来，运行：

sudo phc2sys -s enp1s0f0 -w -m -R 8 -f ./configs/default.cfg

然后您应该看到以下输出：

phc2sys[4348.303]: CLOCK_REALTIME phc offset       -25 s2 freq   +8026 delay   1467
phc2sys[4348.428]: CLOCK_REALTIME phc offset       -11 s2 freq   +8033 delay   1466
phc2sys[4348.553]: CLOCK_REALTIME phc offset       -25 s2 freq   +8016 delay   1396
phc2sys[4348.678]: CLOCK_REALTIME phc offset        -5 s2 freq   +8028 delay   1397

这里的第一个值用于确定 PTP 同步是否正确，为此我们查找 -100 到 100 范围内的值。

在上述两个命令中 enp1s0f0 是 DU 上获取 PTP 同步的网络接口。

使用 DPDK 启用 PTP

如果您在同一接口上将 PTP 和 OFH 与 DPDK 一起使用，则需要使用虚拟功能 (VF) 以确保 PTP 同步正确传递到相关组件。

首先，确定要用于 PTP 和 OFH 的网络端口。您可以使用以下命令列出可用的网络端口：

dpdk-devbind.py -s

您将看到以下输出或类似输出：

Network devices using using kernel driver
=============================================
0000:51:00.0 'Ethernet Controller E810-C for SFP 1593' if=enp81s0f0 drv=ice unused=vfio-pci *Active*
0000:51:00.1 'Ethernet Controller E810-C for SFP 1593' if=enp81s0f1 drv=ice unused=vfio-pci *Active*
0000:51:00.2 'Ethernet Controller E810-C for SFP 1593' if=enp81s0f2 drv=ice unused=vfio-pci
0000:51:00.3 'Ethernet Controller E810-C for SFP 1593' if=enp81s0f3 drv=ice unused=vfio-pci

在上面的输出中， enp81s0f0 是我们要用于 PTP 和 OFH 的网络端口。

接下来，您需要创建 VF，可以通过运行以下命令来完成此操作。记得更换 <INTERFACE NAME> 与您的网络接口的名称（例如， enp81s0f0):

# Remove old VFs
echo 0 | /sys/class/net/<INTERFACE NAME>/device/sriov_numvfs

# Create new VF
echo 1 | /sys/class/net/<INTERFACE NAME>/device/sriov_numvfs

# Set the MAC address for the VF
sudo ip link set <INTERFACE NAME> vf 0 mac 00:33:22:33:00:11 spoofchk off

现在，您可以通过运行以下命令来验证 VF 是否已创建：

dpdk-devbind.py -s

您应该看到以下输出或类似输出：

0000:51:00.3 'Ethernet Controller E810-C for SFP 1593' if=enp81s0f3 drv=ice unused=vfio-pci

在上面的例子中，我们可以看到VF已经创建并且正在使用 iavf 司机在排队 自适应虚拟功能。注意，这是 仅 对于Intel E810网卡，其他网卡可能有不同的驱动程序。

接下来，您需要将 VF 绑定到 vfio-pci 驱动程序，您可以通过运行以下命令来执行此操作。在运行此命令之前，请确保 vfio-pci 已启用，请参阅 DPDK指南 有关如何启用它的更多信息。

sudo dpdk-devbind.py --bind=vfio-pci 0000:51:01.0

srsRAN配置

CU/DU 的示例配置文件可以在 configs srsRAN 源文件的文件夹。每个测试的 RU 都有一个关联的配置文件。

CU/DU 的主要配置步骤发生在 ru_ofh 场。这里，CU/DU 配置为匹配所使用的 RU 的功能。所有参数均应针对每个 RU 进行专门配置。

请参阅以下中的具体 RU 指南 集成指南 有关配置 CU/DU 的更多信息。

鲁

请参阅具体的 RU 文档以正确配置 RU。在尝试进行任何配置更改之前，请确保 RU 正在运行。

我们建议使用制造商文档以及 SRS 的特定 RU 指南（如果在 集成指南.

核心

对于此设置，Open5GS 用作核心，它在 docker 中运行。

Open5GS 5G 核心快速入门指南 全面概述了如何配置 Open5GS 以作为 5G 内核运行。

要正确配置核心，需要执行以下步骤：

  • 配置内核以连接到 gNB，确保两者的 AMF 地址正确。

  • 配置 PLMN 和 TAC 值，使其与 gNB 配置中的值相同。

  • 通过 Open5GS WebUI 将 UE 的 ISIM 凭据注册到订阅者列表。

初始化网络

鲁

要启动 RU，只需启动它并确保其正常运行，然后再尝试连接 DU。检查 RU 同步以及 PTP 进程是否正确运行。

这些具体步骤将根据所使用的 RU 的不同而有所不同。

CU/DU

在运行 CU/DU 之前，请确保您已使用上面配置部分中概述的命令来确认 DU 和前传交换机之间的 PTP 同步。

sudo ./gnb -c <RU_CONFIG>
