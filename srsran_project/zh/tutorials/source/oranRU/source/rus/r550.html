贝尼特 RAN550/RAN650

警告

本文档旨在用作指南。本地设置中固件和软件版本的差异可能需要更改所提供的示例配置文件。因此，请严格遵循您的 RU 的特定用户指南以及本指南。

概述

本指南提供了有关使用 OFH Lib 将 srsRAN CU/DU 连接到 RU 的更多详细信息。具体来说 无线网络550 和 RAN650 来自 Benetel 的 RU。这些分别是 Split 7.2a 室内和室外 RU。

注意事项

请参阅 贝内泰尔 用户指南 最新配置和使用指南的文档，以及免责声明和保修信息。联系贝内特（销售@贝内特尔.com）了解更多信息。

配置

提示

有关最新的配置更改、故障排除和潜在问题的信息，请查看 GitHub 讨论

CU/DU

注意事项

在早期版本中，我们提供了长和短 PRACH 格式的配置示例。 这不再是必需的，所有默认配置现在都使用短 PRACH 格式。

DU 的示例配置文件可以在 configs srsRAN 源文件的文件夹。

这些配置是 具体的 到 RU 的固件版本。在本指南中，我们使用了：

  • RAN550_1_v1.4.1-NM-25fa970 适用于 RAN550

  • RAN650_1_v1.4.0-NM-5561771 适用于 RAN650

以下摘录显示了如何配置 DU 以与 RU 通信（对于 4x2 MIMO 配置）：

ru_ofh:
  t1a_max_cp_dl: 470
  t1a_min_cp_dl: 419
  t1a_max_cp_ul: 336
  t1a_min_cp_ul: 285
  t1a_max_up: 345
  t1a_min_up: 294
  ta4_max: 200
  ta4_min: 0
  is_prach_cp_enabled: true            # Configures if Control-Plane messages should be used to receive PRACH messages.
  compr_method_ul: bfp                 # Uplink compression method.
  compr_bitwidth_ul: 9                 # Uplink IQ samples bitwidth after compression.
  compr_method_dl: bfp                 # Downlink compression method.
  compr_bitwidth_dl: 9                 # Downlink IQ samples bitwidth after compression.
  compr_method_prach: bfp              # PRACH compression method.
  compr_bitwidth_prach: 9              # PRACH IQ samples bitwidth after compression.
  enable_ul_static_compr_hdr: false    # Configures if the compression header is present for uplink User-Plane messages (false) or not present (true).
  enable_dl_static_compr_hdr: false    # Configures if the compression header is present for downlink User-Plane messages (false) or not present (true).
  iq_scaling: 20                       # IQ samples scaling factor applied before compression.
  cells:
    - network_interface: enp1s0f0      # Ethernet interface name used to communicate with the RU.
      ru_mac_addr: 70:b3:d5:e1:5b:06   # RU MAC address.
      du_mac_addr: 80:61:5f:0d:df:aa   # DU MAC address.
      vlan_tag_cp: 5                   # VLAN tag value for CP.
      vlan_tag_up: 5                   # VLAN tag value for UP.
      prach_port_id: [4, 5]            # PRACH eAxC port value.
      dl_port_id: [0, 1, 2, 3]         # Downlink eAxC port values.
      ul_port_id: [0, 1]               # Uplink eAxC port values.
cell_cfg:
  prach:
    prach_root_sequence_index: 1
    prach_config_index: 159
    zero_correlation_zone: 0
  ssb:
    ssb_block_power_dbm: 0 # 0 for RAN650 with 37dBm Tx power, -11 for RAN550 with 24dBm Tx power

为了扩展这一点，在中设置以下参数 cells 字段：

  • network_interface ：用于发送 OFH 数据包的网络接口。

  • ru_mac_addr ：MAC RAN550/650的地址。

  • du_mac_addr ：gNB 使用的接口的 MAC 地址（应直接连接到 RU 或通过支持 PTP 的交换机）。

  • vlan_tag_up/vlan_tag_cp ：V-LAN 标识符，应设置为交换机设置中配置的值

鲁

请参阅 Benetel 用户指南文档来应用以下配置更改，以便它们与您的本地设置相匹配。以下信息仅供参考，可能不适用于您的特定设置。

我们目前建议使用以下 RU 配置，因为其他参数组合未经测试并且似乎 导致 Benetel RU 出现问题：

  • 4x2 MIMO 配置，带宽为 100 MHz

在尝试进行任何配置更改之前，请确保 RU 正在运行。一些 的值可以在中设置/修改 /etc/ru_config.cfg 文件，但其他一些需要修改

  • MAC Address ：DU 的 MAC 地址必须在 RU 中配置用于控制平面和用户平面流量。在我们的配置中，我们对两个平面使用相同的 MAC 地址。

  • VLAN tag ：在我们的设置中，所有网络流量都使用相同的 VLAN ID，因为仅使用一个 MAC 地址。

  • Compression ：配置动态压缩。我们对所有上行链路和下行链路信道使用 BFP9 压缩。有关如何在 RU 中配置压缩的详细信息，请参阅 Benetel 用户指南。

  • Transmission Power ：根据您的设置，您可能需要更改 RU 的传输功率。例如，在 UE 靠近 RU 的实验室设置中，默认功率设置可能会导致 UE 饱和。

  • PRACH format ：我们建议使用短 PRACH 格式。

  • DL scaling ：我们使用 0dB 的下行链路缩放并调整 DU 中的 iq_scaling 参数。

  • TDD pattern ：TDD 模式应设置为 7-2 格式 (DDDDDDDDSUU)。有关其他支持的模式，请参阅 Benetel 指南。

我们用于这些设置的完整配置文件可以在下面找到：

  • 4x2 MIMO configuration with 100 MHz bandwidth.

初始化并连接到网络

初始化和连接到网络的方式与一般 7.2 RU 指南中概述的方式相同。

初始化网络

应采取以下步骤来初始化网络：

  1. 确保 RX50 在线且 PTP 进程和 RU 同步运行正常。

  2. 运行 CU/DU，确保 DU 和 Falcon 交换机之间的 PTP 同步成功，如前所述。

    sudo ./gnb -c gnb_ru_ran550_tdd_n78_100mhz_4x2.yml

如果 DU 成功连接到 RU，您将看到以下输出：

The PRACH detector will not meet the performance requirements with the configuration {Format 0, ZCZ 0, SCS 1.25kHz, Rx ports 1}.
