皮科康

警告

本文档旨在用作指南。本地设置中固件和软件版本的差异可能需要更改所提供的示例配置文件。因此，请严格遵循您的 RU 的特定用户指南以及本指南。

概述

本指南提供了有关使用 ORAN 7.2 split 将 srsRAN CU/DU 连接到 RU 的更多详细信息。具体来说，我们讨论 Picocom PC802SCB。

RU 基于 PC802 小型基站开发板 (PC802SCB) 和 ADI ADRV9029BBCZ 4T4R RFIC 子系统，能够实现高达 100MHz 4T4R。 PC802SCB 是一个灵活的 5G NR 开发平台，用于评估 PC802 的不同性能 小型基站用例，包括拆分 7.2 O-RU。您可以阅读更多相关内容 这里.

注意事项

PC802SCB 不是可部署的商业级 O-RU，但许多商业 O-RU 都基于 PC802 平台。

配置

提示

有关最新的配置更改、故障排除和潜在问题的信息，请查看 GitHub 讨论

CU/DU

DU 的示例配置文件可以从以下位置下载 这里.

以下摘录显示了 DU 如何配置为与 RU 通信：

ru_ofh:
  t1a_max_cp_dl: 350                                              # 下行链路控制平面上的最大 T1a（以微秒为单位）。
  t1a_min_cp_dl: 250                                              # 下行链路控制平面上的最小 T1a（以微秒为单位）。
  t1a_max_cp_ul: 250                                              # 上行链路控制平面上的最大 T1a（以微秒为单位）。
  t1a_min_cp_ul: 150                                              # 上行链路控制平面上的最小 T1a（以微秒为单位）。
  t1a_max_up: 200                                                 # 用户平面上的最大 T1a（以微秒为单位）。
  t1a_min_up: 80                                                  # 用户平面上的最小 T1a（以微秒为单位）。
  ta4_最大: 300                                                    # 用户平面上的最大 Ta4（以微秒为单位）。
  ta4_分钟: 10                                                     # 用户平面上的最小 Ta4（以微秒为单位）。
  is_prach_cp_enabled: 假的                                      # 配置是否应使用控制平面消息来接收 PRACH 消息。
  压缩方法_ul: 巴菲特                                            # 上行链路压缩方法。
  compr_bitwidth_ul: 9                                            # 上行链路IQ采样压缩后的位宽。
  压缩方法_dl: 巴菲特                                            # 下行压缩方式。
  压缩位宽_dl: 9                                            # 下行IQ采样压缩后的位宽。
  compr_方法_普拉赫: 巴菲特                                         # PRACH压缩方法。
  压缩位宽_普拉赫: 9                                         # PRACH IQ 压缩后采样位宽。
  智商缩放: 1.0                                                 # 压缩前应用的 IQ 样本缩放因子，应为小于 10 的正值。
  细胞:
    - 网络接口: enp1s0f0                                 # 用于与RU通信的以太网接口名称。
      ru_mac_地址: ce:fc:6c:09:a6:cd                              # RU MAC 地址。
      杜_mac_地址: 80:61:5f:0d:df:aa                              # DU MAC 地址。
      vlan_tag_cp: 3                                              # C 平面的 VLAN 标记值。
      vlan_tag_up: 3                                              # U 平面的 VLAN 标记值。
      普拉赫端口号: [0]                                          # PRACH eAxC 端口值。
      dl_端口_id: [0]                                             # 下行链路 eAxC 端口值。
      ul_端口_id: [0]                                             # 上行链路 eAxC 端口值。

为了扩展这一点，在中设置以下参数 细胞 字段：

  • 网络接口 ：用于发送 OFH 数据包的网络接口。

  • ru_mac_地址 ：MAC RU的地址。

  • 杜_mac_地址 ：DU 用于 OFH 流量的接口的 MAC 地址。

  • vlan_tag_up/vlan_tag_cp ：VLAN 标识符，应设置为 RU/交换机设置中配置的值。

在 单元格配置 配置部分， 普拉赫 和 tdd_ul_dl_cfg 字段也必须正确配置。以下摘录显示了这一点：

普拉赫:
  普拉赫配置索引: 159                                       # PRACH配置索引。
  prach_root_sequence_index: 1                                  # PRACH根序列索引。
  零相关区: 0                                      # 零相关区域。
  普拉赫_频率_开始: 0                                      # 频域中最低 PRACH 传输时机的 PRB 相对于 PRB 0 的偏移量。
tdd_ul_dl_cfg:
  nof_dl_slots: 7
  nof_ul_插槽: 2

检查这些字段非常重要，因为它们的值可能会根据所使用的 RU 发生变化。

鲁

第一步是打开电路板电源并确保其正确连接到运行 CU/DU 的 PC。接下来，通过 SSH 登录 RU。通过终端正确访问 RU 后， 它可以配置并连接到 CU/DU，OFH 数据包可以在两者之间发送。

注意事项

此配置与固件 v3.0.0 相关。其他固件版本可能需要不同的配置。有关此内容的更多信息，请参阅 Picocom 的相关文档。

配置

必须首先配置 RU，以使 DU 和 RU 的相关值匹配。

RU OFH相关参数可在文件中找到 rf_init_req.json 在这条路上 /home/user/PC-003001-LS/npu/oru_oam/inputs。在这个 JSON 文件中，有一个名为 ecpri_cu_plane_参数:

“ecpri_cu_plane_params” : {
    “is_static_compress” : 1,
    “压缩方法” : 1,
    “压缩iq_宽度” : 9,
    “c_plane_启用”: 1,
    “ul_padding_启用”: 1,
    “fhm_via_sectionid_启用”: 0,
    “cu_mac_check_启用”: 0,
    “eAxC_id_bit_alloc”: 0,
    “du_端口_id” : 0,
    “bandsector_id” : 0,
    “cc_id” : 0,
