srsRAN gNB 与 DPDK

注意事项

本教程假设所使用的机器具有 Intel 处理器。对于 AMD 处理器，使用的一些命令将会改变。总体步骤将基本保持不变。

简介

本教程概述了如何配置 DPDK 以与 srsRAN CU/DU 一起使用。

DPDK代表数据平面开发套件，是一组软件库和驱动程序，用于提高 CU/DU 中数据包处理的性能。

具体来说，以srsRAN CU/DU为例，这可以使用户在某些配置下获得更高的吞吐量和更稳定的性能。例如，需要用户执行以下操作的用例 在 4x4 MIMO 配置中运行 O-RU 可以从使用 DPDK 中获益匪浅。

安装DPDK

注意事项

在本教程中我们将使用 vfio-pci 但你可以 也使用 igb_uio 或 uio_pci_generic。欲了解更多信息 驱动请参考 本文件.

正如 DPDK 文档中所述：

VFIO 内核通常默认存在于所有发行版中，但是请查阅您的发行版文档以确保情况确实如此。

要利用完整的 VFIO 功能，内核和 BIOS 都必须支持并配置为使用 IO 虚拟化（例如 Intel® VT-d）。

在继续之前，请确保您的系统满足上述要求。如果您的系统不满足要求，您可以继续 igb_uio 模块， 如中所述 DPDK 文档.

首先安装以下要求：

sudo apt install build-essential tar wget python3-pip libnuma-dev meson ninja-build python3-pyelftools

建议将 DPDK 23.11 与 srsRAN CU/DU 一起使用。这可以通过安装 以下命令：

wget https://fast.dpdk.org/rel/dpdk-23.11.tar.xz
tar xvf dpdk-23.11.tar.xz dpdk-23.11/
光盘 dpdk-23.11/
meson setup build
光盘 build
ninja
sudo meson install
sudo ldconfig

为了使用 vfio-pci 模块，我们需要在BIOS中启用IOMMU。这通常是默认完成的，但仍应进行检查。 此外，IOMMU需要在内核中激活。根据您的 CPU，您可能需要添加 intel_iommu=开 iommu=pt 为了 英特尔 CPU 或 amd_iommu=开 iommu=pt 对于AMD CPU GRUB_CMDLINE_LINUX_DEFAULT 线路输入 /etc/默认/grub。此后， 更新 grub 并重新启动系统：

sudo update-grub
sudo reboot

下一步是确保 vfio-pci 模块已正确加载。这可以通过以下命令来完成：

sudo modprobe vfio-pci

验证一下 vfio-pci 使用以下命令正确加载：

sudo ./usertools/dpdk-devbind.py -s

您应该看到类似于以下内容的输出：

0000:05:00.0 “用于 SFP 159b 的以太网控制器 E810-XXV” 如果=enp5s0f0 驾驶室=ice 未使用=igb_uio,vfio-pci *Active*

0000:09:00.0 “RTL8125 2.5GbE 控制器 8125” 如果=enp9s0 驾驶室=r8169 未使用=igb_uio,vfio-pci

如果 vfio-pci 模块不存在，它可能存在多个问题，这些问题超出了本教程的范围。 如果是这种情况，请参阅操作系统维护人员或 CPU 供应商的文档以获取更多信息。

您可以继续igb_uio 模块，如有必要，如下所述。

警告

仅当您无法正确加载时才执行此操作 vfio_pci 模块。

您可以安装并加载 igb_uio 使用以下命令：

git clone http://dpdk.org/git/dpdk-kmods
光盘 dpdk-kmods/linux/igb_uio
make
sudo modprobe uio # 确保 uio 模块已加载
sudo insmod igb_uio.ko

使用以下命令确保模块已正确加载：

lsmod | grep uio

您应该看到类似于以下内容的输出：

igb_uio               36864  0
uio                   24576  1 igb_uio

如果模块不存在，请使用 消息 检查潜在错误：

sudo dmesg -T

有关更多信息和故障排除提示，请参阅 DPDK 文档和操作系统维护人员的文档。

配置DPDK

配置大页

DPDK 需要 大页 进行配置才能正确运行。的 dpdk-hugepages.py 可以使用帮助程序脚本来正确配置它。我们建议使用 2G 大页面中的 2GB CU/DU 运行单扇区 4x2 100MHz。如果运行更多扇区，则需要增加大页的数量。

sudo ./dpdk-hugepages.py -p 1G --setup 2G

要使这些更改在整个启动周期中保持不变，请运行以下命令：

sudo mkdir -p /mnt/huge

然后在末尾添加以下行 /etc/fstab:

nodev /mnt/huge hugetlbfs 页面大小=1G 0 0

并编辑此行 /etc/默认/grub:

GRUB_CMDLINE_LINUX_DEFAULT=“安静的飞溅 intel_iommu=on iommu=pt Hugepagesz=1G Hugepages=2 default_hugepagesz=1G”

之后，更新 grub 配置并重新启动系统：

sudo update-grub
sudo reboot

重新启动后，使用以下命令验证是否正确配置了大页：

cat /proc/cmdline
cat /proc/meminfo

EAL参数

DPDK中使用EAL（环境抽象层）参数为常见的环境相关任务（例如内存分配、线程）提供一组函数和抽象 管理和初始化。 DPDK 文档涵盖 EAL 参数 这里.

在 srsRAN 的上下文中，我们使用 实际参数 配置文件中的参数指示 DPDK 哪些内核用于某些进程。如果配置不正确，则 CU/DU 不会利用使用 DPDK 带来的性能改进。这是因为 DPDK 只能看到单个核心，并且无法正确运行并发进程。

EAL 参数，如上述文档中定义，只是作为参数传递给 实际参数 在 CU/DU 配置中。文档中提到的任何参数都可以在 这边。

配置示例如下：

哈尔:
   实际参数: “--lcores'(0-1)@(0-23)'”

这将告诉 DPDK EAL 线程 [0 - 1] 应使用内核 [0 - 23]。假设 CU/DU 在 24 核机器上运行。

另一个例子是：

哈尔:
   实际参数: “--lcores (0-1)@(0-23) -a 0000:52:00.0”

此配置告诉 DPDK EAL 线程 [0 - 1] 应使用内核 [0 - 23] 作为绑定到地址 [0000:52:00.0] 的设备。

与 DPDK 一起运行 srsRAN

如果您还没有这样做，请使用以下命令下载代码库：

git clone https://github.com/srsRAN/srsRAN_Project.git

光盘 build

make -j $(nproc)
make 测试 -j $(nproc)
