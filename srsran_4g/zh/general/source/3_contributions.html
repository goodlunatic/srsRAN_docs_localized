贡献

我们始终欢迎对 srsRAN 4G 软件套件做出贡献。最简单的贡献方法是在 srsRAN 4G 存储库。我们使用 clang-format 来保持一致的代码风格 - 请参阅我们的 默认格式.

我们要求 srsRAN 4G 贡献者同意版权许可协议。这为我们提供了使用贡献代码的必要权限。有关更多信息，请参阅下面的常见问题解答。查看和接受 CLA 协议已集成到 GitHub 拉取请求工作流程中，使用 CLA助理.

常见问题解答

1. 什么是版权许可协议 (CLA)？为什么我需要一份？

版权许可协议是一份法律文件，您在其中声明您有权向您所参与的项目贡献代码/文档/翻译，并愿意将其用于发行版和衍生作品。这意味着，如果将来出现任何关于任何特定代码片段的起源和所有权的法律问题，那么该项目将拥有贡献者提交的必要表格，表明他们被允许做出此贡献。

CLA 还确保，一旦您提供了捐款，您以后就无法尝试撤回其使用许可。因此，人们和公司可以使用该软件，并确信他们以后不会被要求停止使用代码片段。

srsRAN 4G 项目使用的协议是 Project Harmony 提供的标准文档，Project Harmony 是一个以社区为中心的组织，专注于自由开源软件 (FOSS) 的贡献者协议。欲了解更多信息，请访问 www.harmonyagreements.org。

2. 如何填写并提交 CLA？

个人贡献的 srsRAN 4G CLA 可以找到 here。可以找到实体贡献的 srsRAN 4G CLA here。下载适当的 CLA，然后打印、签署并扫描该文档，然后通过电子邮件发送至 许可@软件无线电系统公司.com.

3.我对srsRAN 4G的贡献将如何使用？

srsRAN 4G 项目由总部位于爱尔兰的私人有限公司 Software Radio Systems (SRS) 创建和维护。 SRS 在开源 AGPLv3 许可证和商业许可证下提供 srsRAN 4G。 SRS 还销售基于 srsRAN 4G 代码库构建的专有软件产品。通过这种方式，我们试图确保 srsRAN 4G 项目的持续开发、演进和可持续性。

通过许可协议，我们要求您授予我们在 srsRAN 4G 中使用您的贡献的许可，并继续在开源和商业许可下以及在专有产品中提供 srsRAN 4G。由于我们不要求版权转让，因此您保留对您的贡献的完整所有权，并且拥有与您在未签订许可协议的情况下拥有的相同权利来使用或许可这些贡献。

如果您对 srsRAN 4G 许可和贡献有任何疑问，请通过以下方式联系我们： 许可@软件无线电系统公司.com
