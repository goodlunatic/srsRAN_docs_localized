5G SA srsUE

简介

srsRAN 4G 22.04 版本为 srsUE 带来了 5G SA（独立）支持。 本应用笔记展示了如何将 srsUE 与第三方 5G SA 网络结合使用。在这个例子中， 我们使用 Amarisoft 的 Amari Callbox Classic 来提供网络。

硬件要求

本应用笔记使用了以下组件：

  • Amari Callbox 支持 5G SA 作为 gNB 和核心

  • AMD Ryzen5 3600X Linux PC 作为 UE 计算平台

  • Ettus Research USRP X310 通过 10GigE 连接作为 UE RF 前端

Amari Callbox 是 Amarisoft 推出的基于 LTE/NR SDR 的 UE 测试解决方案。 它包含一个 EPC、一个 5GC、一个 eNodeB、一个 gNodeB、一个 IMS 服务器、一个 eMBMS 服务器和带有 PCIe SDR 卡的 Intel i7 Linux PC。 gNodeB 符合版本 15，并且 支持 NSA 和 SA 模式。规格的进一步概述可以在 数据表。 选择该测试解决方案是因为它广泛可用、易于配置且用户友好。

硬件设置

测试可以通过无线方式或使用有线设置进行。 在此示例中，我们在 UE 和 eNB/gNB 之间使用电缆设置（即从 X310 到 PCIe SDR 卡） 在呼叫箱上）。这些连接通过 30dB 衰减器，如上图所示。的 还显示了用于 UE 和 Callbox 精确时钟的 PPS 输入。 UE 和 Callbox 都需要精确的时钟 - 在我们的测试中，我们为两者提供 PPS 输入。

局限性

当前的 5G SA UE 应用程序有一些功能限制，需要某些配置 gNB 和核心网络的设置。主要功能限制如下：

  • 子载波间隔 (SCS) 限制为 15 kHz，这意味着只能使用 FDD 频段。

  • 限制为 5、10、15 或 20 MHz 带宽 (BW)

配置

要设置和运行 5G、SA 网络和 UE，两个网络的配置文件 Callbox 和 srsUE 必须更改。

所有修改的配置文件已作为本应用说明的附件包含在内。它是 建议您使用这些文件以避免手动更改配置时出现错误。任意配置 此处未包含的文件不需要对默认设置进行修改。

srsUE

需要对 UE 配置文件进行以下更改以允许其连接 SA 模式下的 Callbox。

首先需要更改以下参数 [射频] 选项，以便 X310优化配置：

[射频]
发送增益 = 3
频率偏移 = 0

nof_天线 = 1

斯拉特 = 11.52e6

设备名称 = 超高清
设备参数 = 类型=x300,连续剧=30B8658,时钟=外部的,采样率=23.04e6,lo_freq_offset_hz=23.04e6,无

需要进行下一组更改[大鼠.eutra] 选项。 LTE 载波已禁用，以强制 UE 使用 5G NR 载波：

[老鼠.尤特拉]
dl_earfcn = 2850
nof_运营商 = 0

最后 [大鼠.nr] 5G SA 模式操作需要配置选项：

[老鼠.号]
nof_运营商 = 1
乐队 = 3

电话亭

的 amarisoft_enb.cfg 文件负责 5G SA 网络所需的 gNB 的配置。

默认配置的主要变化如下：

  • 启用 NR 支持

  • 启用 NR 小区并配置 NR 小区

  • 修改PRACH配置

  • 修改PUCCH配置

启用 NR 支持

这是在第 47 行通过将相应的标志设置为 true 来完成的：

号码支持: 真实,

NR细胞

首先必须设置 Band 和 ARFCN。这是在第 61 和 62 行完成的：

编号单元列表: [
{
  射频端口: 0,
  小区编号: 1,
  乐队: 3,
  dl_nr_arfcn: 368500,
},

],

的 乐队 和 dl_nr_afcn 根据 srsRAN 4G 的已知限制进行选择。

接下来，从第68行开始可以更改SCS、BW等配置参数：

nr_cell_默认值: {
  子载波间距: 15, /* 千赫兹 */
  ssb_子载波_间距: 15, // 仅 支持 在 FDD 乐队
  带宽: 10, /* 兆赫兹 */
  n_天线_dl: 1,
  n_天线_ul: 1,


  ssb_pos_位图: "1000",

  ssb_周期: 10, /* 在 女士 */
  n_id_cell: 500,

这里的 子载波间距 设置为 15 KHz 并且 带宽 至 10 MHz， n_天线_dl 设置为 1 并且 ssb_周期 设置为 10。

普拉奇

对于 PRACH 配置选项（第 105 行），使用以下内容：

普拉赫: {
  普拉赫配置索引: 0,
  消息1_fdm: 1,
  msg1_频率_开始: 1,
  零相关区域配置: 0,
  前导码接收目标功率: -110, /* 在 分贝 */
  前导码传输最大: 7,
  功率斜坡步长: 4, /* 在 分贝 */
  ra_response_window: 10, /* 在 插槽 */
  限制集配置: “无限制集”,
  ra_contention_resolution_timer: 64, /* 在 女士 */
  ssb_per_prach_occasion: 1,
  cb_preambles_per_ssb: 8,
},

对上述内容所做的更改包括设置 普拉赫配置索引 为 0，设置 msg1_频率_开始 至 1 并设置 ra_response_window 到 10。

公共信道信道

最后，必须更改 PUCCH 配置。这是从第 353 行开始完成的：

普奇: {
  pucch_group_hopping: “都不是”,
  跳跃ID: -1, /* -1 = n_cell_id */
  p0_标称值: -90,
    普奇1: {
      n_cs: 3,
      n_occ: 3,
      跳频: 假的,
    },
    普奇2: {
      n_符号: 2,
      n_prb: 1,
      跳频: 假的,
      同时_harq_ack_csi: 假的,
      最大码率: 0.25,
    },
},

这里唯一的变化是 跳频 在 pucch1 和 pucch2 中都设置为 false。

gNB 现已正确配置。与 gNB 和 5GC 关联的所有其他配置文件都可以保留默认状态。

运行网络

运行网络时应使用以下顺序：

  1. 5GC

  2. gNB

  3. UE

5GC

要运行 5GC，请使用以下命令：

须藤 尔泰梅 嗯.cfg

gNB

接下来应使用以下命令实例化 eNB/gNB：

须藤 勒滕布 国民银行-国家安全局.cfg

控制台输出应类似于：

      基地 车站 版本 2021-03-15, 版权 (C) 2012-2021 阿玛瑞软件

射频0: 采样率=61.440 兆赫兹 dl_频率=1836.740 兆赫兹 ul_频率=1741.740 兆赫兹 (乐队 n3) dl_ant=1 ul_ant=1

UE

要运行 UE，请使用以下命令：

须藤 斯苏埃 厄.会议

UE 初始化后，您应该看到以下内容：

接下来是有关 USRP 的一些信息。成功找到单元格后，您应该看到以下内容：

找到了 细胞:  模式=FDD, PCI=1, PRB=50, 港口=1, 首席财务官=0.1 千赫兹

随机 访问 传输: 普拉赫_场合=0, 前导码索引=0, 拉-rnti=0xf, 蒂=3851
随机 访问 完成.     c-rnti=0x4601, 塔=3
RRC 已连接

配电单元 会议 设立 成功的. 知识产权: 192.168.4.2
RRC NR 重新配置 成功的.

要确认 UE 已成功连接，您应该在控制台输出中看到以下内容 gNB:
