5G SA 现货 UE

提示

srsENB 支持 NSA 和 SA 操作模式的原型 5G 功能，但是，这些功能不再处于积极开发中。相反，我们推荐 srsRAN Project，我们的 ORAN-native 5G CU/DU 解决方案。

警告

在蜂窝频段上运行专用 5G SA 网络可能会受到您所在辖区的严格监管。寻求批准 在此之前先咨询您的电信监管机构。

srsRAN 4G 22.04 为 srsUE 和 srsENB 提供 5G SA 支持。 5G SA 功能可以通过 srsUE 和 srsENB 的配置文件。

本应用笔记演示如何配置支持 5G 的 COTS UE 并将其连接到 5G SA 网络 使用 srsRAN 4G 和第 3 方内核（本例中为 Open5GS）。

网络和硬件概述

设置 5G SA 网络并连接 5G COTS UE 需要以下内容：

  • 具有基于 Linux 的操作系统的 PC，安装并构建了 srsRAN 4G 22.04（或更高版本）

  • 第三方 5G 核心

  • 兼容的 SDR（USRP、LimeSDR、BladeRF 等），理想连接到 10 MHz 参考时钟或 GPSDO

  • A 5G SA 功能 UE

  • USIM/SIM 卡（必须是测试卡或可编程卡，且密钥已知）

对于此实施，使用以下设备：

  • 配备 Ubuntu 20.04 LTS 的戴尔 XPS13

  • srsRAN 4G 22.04 & Open5GS

  • 通过 USB3 运行的 B200-mini USRP

  • 配备 Sysmocom USIM 的 OnePlus Nord 5G

UE 注意事项

srsRAN 4G 当前的限制之一是仅支持 15 kHz 子载波间隔。因此，只能使用能够在支持 FDD 的频段（例如频段 3）中运行的设备。 许多商业 COTS UE 要求 TDD 频段使用 30 kHz SCS，但我们目前不支持。

除了来自基带硬件的限制之外，还有一些其他陷阱可能会或可能不允许手机连接到 5G 网络：

  • 在某些手机上，使用测试 USIM 时，您可能需要使用以下命令激活 5G NR*#*#4636#*#*.

  • 如果您的手机支持“Smart 5G”，请禁用此选项，因为它可能会强制手机设置为 4G 并激活漫游。

  • 许多 5G 手机可能包含运营商策略文件，该文件可能会根据 USIM 的 PLMN（IMSI 的前 6 位数字）限制电话的 5G 功能。运营商策略文件通常不包括测试网络 PLMN，因此设置测试 PLMN 可能会导致 5G 被禁用。如果可能，使用屏蔽盒并通过商业运营商 PLMN 配置网络可以避免策略文件问题。

RF驱动程序

检查运行时是否已拾取 RF 驱动程序 cmake .. 在构建过程中，您可以运行：

grep <驱动程序> srsRAN_4G/build/CMakeCache.txt

如果您使用 UHD 作为驱动程序，则当 srsRAN 4G 成功检测到它时，您应该会看到以下输出 cmake .. 运行：

$ grep UHD -m 4 CMakeCache.txt

//Enable UHD
ENABLE_UHD:BOOL=ON
UHD_INCLUDE_DIRS:PATH=/usr/local/include
UHD_LIBRARIES:FILEPATH=/usr/local/lib/libuhd.so

我们仅使用 Ettus Research 设备测试了 SA 模式 超高清。对于本应用说明，我们使用 UHD 版本 v4.2 的 USRP B200-mini。

srsRAN 4G

如果您尚未这样做，请安装最新版本的 srsRAN 4G 及其所有依赖项。这在 安装指南.

注意事项

如果您安装或更新驱动程序 之后 安装 srsRAN 4G 后，您将必须重新构建 srsRAN 4G。

第 3 方核心 - Open5GS

在本示例中，我们使用 Open5GS 作为 5G 内核。

Open5GS 是 5G 核心和 EPC 的 C 语言开源实现。以下链接将为您提供 包含下载和设置 Open5GS 所需的信息，以便可以与 srsRAN 4G 一起使用：

  • GitHub

  • 快速入门指南

srsENB 将通过以下方式连接到 AMF 和 UPF 内存地址 srsENB 配置文件中的配置选项。

准备 COTS UE

大多数 COTS UE 连接到 srsRAN 4G 不会有任何问题，并且应该可以开箱即用。如果您在查看或连接网络时遇到困难，可以执行以下步骤 需要遵循：

  • 5G USIM 的正确配置

  • 生根UE

  • 使用 Network Signal Guru 强制设备看到 5G 单元

以下步骤不是运行 5G SA 所必需的，但如果您在连接网络时遇到问题，则可能需要执行以下步骤。

5G SIM 卡

如果您使用的是支持 5G 的 sysmcom-ISIM，则需要修改 SIM 卡的 5G 相关字段。特别是您需要启用 SUCI 隐藏。这可以通过 SIM 卡读卡器使用以下命令来完成（首先，您需要将 ADM 引脚添加到 ./scripts/deactivate-5g.script）：

./模拟仿真-外壳.py -p0 --脚本 ./脚本/停用-5克.脚本

您可以在以下位置找到更多相关信息： 本指南，由梅林·克洛斯塔撰写。

生根COTS UE

生根后将允许您运行 Network Signal Guru (NSG) 应用程序。它还允许您配置系统设置并授予对标准使用不允许的其他功能的访问权限。

如何完成此操作取决于您的设备的品牌和制造商。 XDA-Developers 有一篇有用的文章，概述了如何 root 各种 COTS UE 设备，您可以找到它 这里.

警告

生根设备可能会导致您丢失设备上存储的所有信息。不建议对您的个人设备进行root。在进行此过程之前，您应该小心地充分了解您正在做什么。

网络信号大师

注意事项

为了使 NSG 正常工作并强制连接到 5G SA 单元，您的设备必须具有 Qualcomm 基带处理器并且也已获得 root 权限。

您可以从 Play 商店下载 NSG： 链接.

NSG 将用于强制 COTS UE 使用特定 RAT，在本例中为 5G SA。为此，请执行以下步骤：

  • 在应用程序的右上角，按 3 个点打开下拉菜单。

  • 现在选择 强迫 控制.

  • 下 “首选网络类型” 选择 仅 5GNR.

  • 在“NR5G”模式下选择 SA.

要激活 LTE 和 NSA，请执行以下步骤：

  • 在“首选网络类型”下选择 5GNR 和 LTE

  • 在“NR5G”模式下选择 NSA/SA

其余设置应保持原样，要么不设置，要么保留默认状态。下面的屏幕截图显示了这一点。

一旦网络启动并运行，您应该能够从应用程序中选择它。这将迫使 UE 连接到它。

配置

为此应用笔记修改了以下配置文件：

  • enb配置文件

  • 日志配置文件

  • amf.yaml

  • upf.yaml

注意事项

22.04.1 对 rr.conf 进行了更改， coreset0_idx 现在是必填字段 nr_nell_列表。如果您使用的是最新更新的旧配置文件，则需要将其添加到您的配置文件中。

以下各节将概述所做修改的详细信息。

血清ENB

要将 srsENB 配置为连接到 5GC 和 COTS UE，需要进行以下更改：

  • enb配置文件

  • 日志配置文件

enb配置文件

首先，需要更改MCC和MNC以匹配Open5GS使用的MCC和MNC，mme_addr也需要 设置为允许 RAN 连接到 AMF 和 UPF。

下图显示了这些修改：

[恩布]
enb_id = 0x19B
MCC = 901
跨国公司 = 70
内存地址 = 127.0.0.2
gtp_bind_地址 = 127.0.1.1
s1c_bind_addr = 127.0.1.1
s1c_bind_端口 = 0
n_prb = 50
#tm = 4
#nof_端口 = 2

srsENB 将自动选择连接的 SDR，在本例中为 B200-mini USRP。进一步 可以使用特定设备参数进行配置。对于本示例，使用了以下配置：

[射频]
#dl_earfcn = 3350
发送增益 = 30
接收增益 = 40

设备名称 = 汽车

如果 UE 无法查看或连接到网络，则可以在此处调整 tx 和 rx 增益值。 RF 信号强度 受与每个用例和设置相关的各种物理条件的影响。因此，上述配置可能无法完美运行 对于所有用户。因此，应根据需要修改它们的配置。

日志配置文件

需要修改 rr.conf 文件以将 NR Cell 添加到小区列表中。默认的 LTE 单元也需要是 注释掉，或从列表中完全删除。 NR Cell 的配置方式如下：

编号单元列表 =
(
  {
    射频端口 = 0;
    小区编号 = 1;
    根序列idx = 1;
    塔克 = 7;
    PCI = 500;
    dl_arfcn = 368500;
    coreset0_idx = 6;
    乐队 = 3;
  }
);

在随附的示例配置中，LTE 单元列表已被简单地注释掉。尽管该列表也可以被删除，或者留空。

核心

如上所述，Open5GS 5G 核心快速入门指南 全面概述了如何配置 Open5GS 以作为 5G 内核运行。

需要进行的主要修改是：

  • 将 AMF 配置中的 TAC 更改为 7

  • 检查NGAP和GTPU地址是否正确。这是在 AMF 和 UPF 配置文件中完成的。

  • 确保上述所有文件和 UE 配置文件中的 PLMN 值一致也是一个好主意。

最后一步是通过 Open5GS WebUI 将 UE 注册到订阅者列表中。每个字段的值应与所使用的 USIM 关联的值相匹配。 这些通常由 USIM 制造商提供。

注意事项

确保正确配置 APN，如果配置不正确，UE 将无法连接。

将 APN 添加到 COTS UE

必须将 APN 添加到 COTS UE 才能使其连接到互联网。此 APN 必须与 Core 中订阅者条目中定义的相同。

默认情况下，当订户通过 WebUI 向 Open5GS Core 注册时，会为其提供包含以下详细信息的 APN：

  • 接入点名称： 互联网

  • APN 协议： IPv4

这是通过 UE 的网络设置完成的。通常通过以下路径（或类似路径）找到：

  • WiFi 和网络 > SIM 和网络设置 > SIM > 接入点名称

然后，应将具有上述凭据的 APN 添加到列表中。

连接到网络

核心

按照上述步骤和 Open5Gs 快速入门指南配置核心后，重新启动 AMF 和 UPF 守护程序非常重要。 每当对任何一个关联的配置文件进行修改时都应该执行此操作，以便所做的任何更改都能生效。

核心不需要直接启动，因为它默认在后台运行。 srsENB 将在启动时自动连接到它。

血清ENB

首先运行 srsENB。在此示例中，srsENB 直接从构建文件夹运行，配置文件也位于此处：

须藤 ./斯尔森布 恩布.会议

故障排除

MCS

NR 调度器当前的局限性之一是缺少动态 MCS 自适应。因此，固定MCS用于下行链路(PDSCH)和上行链路(PUSCH)传输。 默认情况下，我们使用 MCS 28 的最大值作为最大速率。然而，根据 RF 条件，该值可能太高。在这种情况下，尝试使用较低的 MCS，例如：

[调度程序]
nr_pdsch_mcs = 10
nr_pusch_mcs = 10

参考时钟

如果您遇到手机找不到手机和/或无法保持连接的问题，可能是由于 gNB RF 前端的时钟不准确。尝试使用外部 10 MHz 参考或使用 GPSDO 规范的振荡器。

局限性

带宽

目前 srsENB 在 5G SA 模式下运行时仅支持 10 MHz 带宽。

FDD 表带

目前，srsRAN 4G 仅支持 5G SA 使用 FDD 频段。这是因为 srsRAN 4G 仅支持 15 kHz SCS。 自 srsRAN 4G 22.04.1 起，固定 CoreSet0 索引已被删除，现在可以自由配置。然而，并非所有组合 ARFCN 和 CoreSet0 索引是有效配置。如果组合无效，配置值检查器将通知用户 已提供。如果发生这种情况，请更改 ARFCN 或 coreset0_idx 值在 日志配置文件.
