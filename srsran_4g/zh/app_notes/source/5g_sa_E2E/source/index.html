5G SA 端到端

提示

srsENB 支持 NSA 和 SA 操作模式的原型 5G 功能，但是，这些功能不再处于积极开发中。相反，我们推荐 srsRAN Project，我们的 ORAN-native 5G CU/DU 解决方案。

srsRAN 4G 22.04 为 srsUE 和 srsENB 提供 5G SA 支持。 5G SA 功能可通过 srsUE 和 srsENB 的配置文件。本应用笔记演示了如何配置 5G SA 网络 使用 srsRAN 4G、第三方核心 (Open5GS) 和 ZeroMQ (ZMQ)。使用 ZMQ 意味着不需要物理 RF 硬件。

网络和硬件概述

本应用笔记将使用以下硬件和软件：

  • 配备 Ubuntu 20.04.4 的戴尔 XPS13

  • srsRAN 4G 22.04（或更高版本）

  • ZeroMQ

  • Open5GS 5G 核心

有关安装 ZMQ 并将其与 srsRAN 4G 一起使用的信息，请参阅我们的 ZMQ应用说明.

Open5GS

Open5GS 是 5G 核心和 EPC 的 C 语言开源实现。以下链接将为您提供 包含下载和设置 Open5GS 所需的信息，以便可以与 srsRAN 4G 一起使用：

  • GitHub

  • 快速入门指南

配置

要启用 5G SA 功能，需要对 UE 和 eNB 配置文件进行更改。本节将概述这些内容。

示例配置附于此处：

  • ue.conf

  • rr.conf

  • enb.conf

  • amf.yaml

  • upf.yaml

注意事项

22.04.1 对 rr.conf 进行了更改， coreset0_idx 现在是必填字段 nr_nell_列表。如果您使用的是最新更新的旧配置文件，则需要将其添加到您的配置文件中。

这些配置文件已被修改，以删除对此用例并不重要的某些选项。

srsUE

需要执行以下步骤来修改 srsUE 配置文件以启用 5G SA 功能：

  • 启用ZMQ

  • 启用 NR 功能

  • 配置 USIM 凭据和 APN

ZMQ

要启用 ZMQ，请将以下内容添加到 UE 配置文件中：

[rf]
freq_offset = 0
tx_gain = 80
srate = 11.52e6

device_name = zmq
device_args = tx_port=tcp://*:2001,rx_port=tcp://localhost:2000,id=ue,base_srate=11.52e6

注意事项

5G SA 必须使用固定采样率

还必须配置网络命名空间：

[gw]
netns = ue1

降噪特性

首先禁用 LTE 运营商：

[rat.eutra]
dl_earfcn = 2850
nof_carriers = 0

启用 NR 频段和载波：

[rat.nr]
bands = 3,78
nof_carriers = 1

最后，将版本设置为release-15：

[rrc]
release = 15

USIM 凭证和 APN

使用以下 USIM 凭证：

[usim]
mode = soft
algo = milenage
opc  = 63BFA50EE6523365FF14C1F45F88737D
k    = 00112233445566778899aabbccddeeff
imsi = 901700123456780
imei = 353490069873319

这里的主要变化是调整 IMSI，以便使用正确的 PLMN。

通过以下配置启用 APN：

[nas]
apn = srsapn
apn_protocol = ipv4

网络命名空间

使用 ZMQ 时，为 UE 创建适当的网络命名空间非常重要。

要创建网络命名空间，请使用：

sudo ip netns add ue1

要验证新的“ue1”网络是否存在，请运行：

sudo ip netns list

关于为什么需要这样做的更多信息可以在 ZMQ应用说明.

srsENB

需要执行以下步骤来修改 srsENB 配置和关联的配置文件以启用 5G SA 功能：

  • enb配置文件

    • 设置正确的 PLMN

    • 更改 MME 地址以匹配 Open5GS GTPU 和 NGAP 地址”。

    • 启用ZMQ

  • rr.conf

    • 将 5G 单元添加到单元列表，并删除 LTE 单元

enb配置文件

PLMN和MME

设置 PLMN（MCC 和 MNC）和 MME 地址按以下方式完成：

[enb]
enb_id = 0x19B
mcc = 901
mnc = 70
mme_addr = 127.0.0.2
gtp_bind_addr = 127.0.1.1
s1c_bind_addr = 127.0.1.1
s1c_bind_port = 0
n_prb = 50

MMC 和 MNC 设置为与 UE 和 Core 匹配。配置MME地址是为了让eNB能够与AMF和UPF正确通信。如果不更改 srsENB，Core 将无法连接。

ZMQ

通过对配置文件进行以下更改来启用 ZMQ：

[rf]
rx_gain = 40
tx_gain = 80

# Example for ZMQ-based operation with TCP transport for I/Q samples
device_name = zmq
device_args = fail_on_disconnect=true,tx_port=tcp://*:2000,rx_port=tcp://localhost:2001,id=enb,base_srate=11.52e6

rr.conf

在 5G SA 模式下运行时，必须将 5G NR 单元添加到 rr.conf 中，必须删除现有的 LTE 单元。这可以通过注释掉它们或完全删除它们来完成。在附加的 rr.conf 文件中，他们有 已被注释掉。

使用以下 5G NR 单元配置：

nr_cell_list =
(
  {
    rf_port = 0;
    cell_id = 1;
    root_seq_idx = 1;
    tac = 7;
    pci = 500;
    dl_arfcn = 368500;
    coreset0_idx = 6;
    band = 3;
  }
);

核心

如上所述，Open5GS 快速入门指南 全面概述了如何配置 Open5GS 以作为 5G 内核运行。

需要进行的主要修改是：

  • 将 AMF 配置中的 TAC 更改为 7

  • 检查NGAP和GTPU地址是否正确。这是在 AMF 和 UPF 配置文件中完成的。

  • 确保上述所有文件和 UE 配置文件中的 PLMN 值一致也是一个好主意。

最后一步是通过 Open5GS WebUI 将 UE 注册到订阅者列表中。每个字段的值应与 [USIM] 部分下的 UE 配置文件中的值匹配。

注意事项

确保正确配置 APN，如果配置不正确，UE 将无法连接到互联网。

设置网络

核心

遵循 Open5GS 快速入门指南中的步骤后，您无需执行任何其他操作即可使核心联机。它将在后台运行。确保在之后重新启动相关守护进程 对配置文件进行任何更改。

首先运行 srsENB。在此示例中，srsENB 直接从构建文件夹运行，配置文件也位于此处：

测试网络

平

这是测试网络的最简单方法。这将测试UE和核心是否可以成功通信。

上行链路

要测试上行链路方向的连接，请使用以下命令：

sudo ip netns exec ue1 ping 10.45.0.1

下行

要在下行链路方向运行流量，请使用：

ping 10.45.0.2

UE 的 IP 可以从 UE 控制台输出中获取。每次 UE 重新连接到网络时，该值都会发生变化，因此最佳实践是始终通过读取来仔细检查分配的最新 IP 在运行下行链路流量之前从控制台。

性能3

在此设置中，客户端将在 UE 端运行，服务器在网络端运行。 UDP 流量将以 10Mbps 的速度生成，持续 60 秒。运行 iPerf 客户端时，我们使用 UE 网络命名空间并指定网络侧IP地址。首先启动服务器，然后启动客户端，这一点很重要。

网络侧

启动 iPerf 服务器：

iperf3 -s -i 1

然后，它将侦听来自 UE 的流量。

UE-侧

网络和 iPerf 服务器启动并运行后，可以使用以下命令从 UE 的网络命名空间运行客户端：

sudo ip netns exec ue1 iperf3 -c 10.45.0.1 -b 10M -i 1 -t 60

现在流量将从 UE 发送到 eNB。这将显示在服务器和客户端控制台中，也会显示在 UE 和 eNB 的跟踪中。
