载波聚合

简介

在动手之前，我们建议您阅读以下内容 载波聚合.

srsRAN 4G 软件套件支持 srsENB 和 srsUE 中的 2 载波聚合。要使用 srsRAN 4G 无线进行载波聚合实验，您需要一个能够在不同通道中调谐不同频率的 RF 设备，例如 Ettus Research (NI) 的 USRP X300 系列。我们已经使用 UHD 3.15 LTS 和 UHD 4.0 进行了测试。

或者，使用我们基于 ZeroMQ 的 RF 层仿真在没有 SDR 硬件的情况下进行载波聚合实验。看看我们的 ZeroMQ 应用说明 有关 RF 层仿真的更多信息。

使用 SDR 硬件进行载波聚合

eNodeB 配置

要配置 eNodeB 进行载波聚合，我们必须首先配置 RF 前端。然后，我们必须为多个单元配置 srsENB 并定义它们之间的主/从关系。

如果您使用的是真正的 RF 器件，例如 USRP X300，建议使用外部时钟参考，或者使用 10 MHz/1 PPS 输入（clock=external）或 GPSDO（clock=gpsdo）。 对于 X300，尤其是较新的 UHD 版本，还需要在无线电初始化时指定特定的采样率。例如，如果您计划使用 10 MHz 小区 (50 PRB) 无线电的采样率为 11.52 Msamples/s，因此 sampling_rate=11.52e6 应使用。对于 20 MHz 小区 (100 PRB)，采样率为 23.04 Msamples/s，因此sampling_rate=23.04e6 应使用。

[rf]
device_name = uhd
device_args = type=x300,clock=external,sampling_rate=23.04e6

第二步是使用两个单元配置 srsENB。为此，需要修改 rr.conf:

cell_list =
(
  {
    rf_port = 0;
    cell_id = 0x01;
    tac = 0x0007;
    pci = 1;
    root_seq_idx = 204;
    dl_earfcn = 2850;

    // CA cells
    scell_list = (
      {cell_id = 0x02; cross_carrier_scheduling = false; scheduling_cell_id = 0x01; ul_allowed = true}
    )
  },
  {
    rf_port = 1;
    cell_id = 0x02;
    tac = 0x0007;
    pci = 4;
    root_seq_idx = 268;
    dl_earfcn = 3050;

    // CA cells
    scell_list = (
      {cell_id = 0x01; cross_carrier_scheduling = false; scheduling_cell_id = 0x02; ul_allowed = true}
    )
  }
)

完成这些更改后，只需照常运行 srsENB 即可。

UE 配置

在UE中，我们必须再次设置RF配置并配置UE功能。

对于RF配置，我们需要设置列表 根据 eNodeB 中配置的单元，设置 EARFCN 载体数量至 2：

[rf]
dl_earfcn = 2850,3050
nof_carriers = 2

在列表中添加更多 EARFCN 会使 UE 扫描这些频率 并且载波的数量使得UE使用更多的RF通道。

对于 UE 功能，我们至少需要报告版本 10 类和 7 类：

[rrc]
ue_category        = 7
ue_category_dl     = 10
release            = 10

完成这些更改后，只需照常运行 srsUE 即可。

使用 ZeroMQ RF 仿真进行载波聚合

要使用 ZeroMQ RF 仿真而不是 SDR 硬件来试验载波聚合， 我们只需配置 srsENB 和 srsUE 即可使用zmq RF 设备。

eNodeB 配置

对于 srsENB，配置 zmq RF器件如下：

[rf]
device_name = zmq
device_args = fail_on_disconnect=true,id=enb,tx_port0=tcp://*:2000,tx_port1=tcp://*:2002,rx_port0=tcp://localhost:2001,rx_port1=tcp://localhost:2003

UE 配置

对于 srsUE，配置 zmq RF器件如下：

[rf]
device_name = zmq
device_args = tx_port0=tcp://*:2001,tx_port1=tcp://*:2003,rx_port0=tcp://localhost:2000,rx_port1=tcp://localhost:2002,id=ue,tx_freq0=2510e6,tx_freq1=2530e6,rx_freq0=2630e6,rx_freq1=2650e6

由于 ZMQ 模块与频率无关，因此 Tx 和 Rx 频率在 ZMQ 配置中设置。这使得内部载体 切换可能。

已知问题

  • eNodeB 忽略 UE 的频段功能

  • CPU 饥饿且实时错误超过 10 MHz
