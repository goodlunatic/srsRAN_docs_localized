eMBMS 端到端

简介

增强型多媒体广播多播服务 (eMBMS) 是LTE的广播模式。使用 eMBMS，eNodeB 可以有效地向连接到小区的所有用户广播相同的数据。 srsRAN 4G支持端到端系统中的eMBMS，包括srsUE、srsENB和srsEPC。除了这些之外，还有一个新的应用程序 推出了-srsMBMS。 srsMBMS 是 SRS MBMS 网关，是一个额外的网络组件，用于接收多播数据 TUN虚拟网络接口并将其提供给eNodeB中的eMBMS承载。

设置

要使用 eMBMS 运行端到端 srsRAN 4G 系统，需要对 srsENB 和 srsUE 应用程序进行一些额外配置。 在提供的示例配置中，假设 srsmbms、srsepc 和 srsenb 在同一台物理计算机上运行。

srsENB配置

在eNodeB上，需要额外的配置才能支持eMBMS传输。首先，不要使用默认的 sib.conf.example，替代方案 sib.conf.mbsfn.example 应该使用。此版本的 sib 配置添加了 eMBMS SIB2 的参数，并包括特定于 eMBMS 的 SIB 13。这些 SIB 修改定义了以下关键 eMBMS 网络 参数：

  • eMBMS子帧分配

  • MCCH调度周期

  • MCCH 调制顺序

  • 非eMBMS子帧区域长度

  • eMBMS 区域 ID

  • MCCH子帧分配

  • MCCH 重复周期

除了使用 eMBMS SIB 配置文件之外，还必须在 enb.conf:

[enb_files]
sib_config = sib.conf.mbsfn

[embms]
enable = true

[scheduler]
min_nof_ctrl_symbols = 2
max_nof_ctrl_symbols = 2

[expert]
nof_phy_threads = 2

套装 m1u_if_addr 到像这样的本地主机地址 127.0.1.201 或 srsMBMS 二进制文件可用的网络的本地 IP。完成这些设置调整后，eNodeB 应准备好在 eMBMS 模式下运行。

srsMBMS配置

仅当 eMBMS 二进制文件和 eNodeB 在不同机器上运行时才需要调整 eMBMS 配置。参数必须设置在 mbms.conf:

[mbms_gw]
m1u_multi_if   = 127.0.1.200

套装 m1u_if_addr 到像这样的本地主机地址 127.0.1.200 或 eNodeB 可用的网络的本地 IP。

srsUE配置

对于 UE，将从下行链路信号中存在的 SIB 和 MCCH 自动检测 eMBMS 传输的存在。要接收活动的 eMBMS 服务，必须在中设置以下参数 ue.conf:

[rrc]
mbms_service_id = 0

请注意，此服务 ID 必须与网络使用的服务 ID 匹配。

此外，我们建议使用以下设置以获得 eMBMS 的最佳性能：

[phy]
interpolate_subframe_enabled = true
snr_estim_alg = empty
nof_phy_threads = 2

完成这些配置后，您的 UE 应准备好运行 eMBMS。

用途

首先，在同一台计算机上运行 srsMBMS（MBMS 网关）、srsEPC 和 srsENB：

sudo ./srsmbms ~/.config/srsran_4g/mbms.conf

sudo ./srsepc ~/.config/srsran_4g/epc.conf

sudo ./srsenb ~/.config/srsran_4g/enb.conf

MBMS 网关将创建一个 TUN 接口，所有用于多播的流量都应写入该接口。然后，它将通过专用于 eMBMS 流量的单独 GTPU 隧道将此流量转发到 eNodeB。

要使用 srsMBMS、srsEPC 和 srsENB 测试 eMBMS，我们可以使用 伊珀夫。在MBMS网关处，创建路由并启动下行流量：

sudo route add -net 239.255.1.0 netmask 255.255.255.0 dev sgi_mb

iperf -u -c 239.255.1.1 -b 10M -T 64 -t 60

接下来，我们可以在单独的机器上运行 srsUE 来接收 eMBMS 数据：

sudo ./srsue ~/.config/srsran_4g/ue.conf

在启用了 eMBMS 的 eNodeB 上运行 srsUE 后，您应该在 UE 的终端看到以下输出：

Searching cell in DL EARFCN=3400, f_dl=2685.0 MHz, f_ul=2565.0 MHz
Found Cell:  Mode=FDD, PCI=1, PRB=50, Ports=1, CFO=-0.0 KHz
Found PLMN:  Id=00101, TAC=7
Random Access Transmission: seq=20, ra-rnti=0x2
Random Access Complete.     c-rnti=0x46, ta=1
RRC Connected
MBMS service started. Service id:0, port: 4321
Network attach successful. IP: 172.16.0.2
Software Radio Systems LTE (srsRAN  4G)

的 MBMS服务启动。服务ID：0，端口：4321 表示eMBMS服务已成功启动。

要接收组播 iperf 数据，请添加到 UE 的路由并启动 iperf 服务器：

sudo route add -net 239.255.1.0 netmask 255.255.255.0 dev tun_srsue

iperf -s -u -B 239.255.1.1 -i 1
