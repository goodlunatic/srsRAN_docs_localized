开始使用

要开始使用 srsEPC，您需要一台装有基于 GNU/Linux 操作系统的 PC。 这可以是您喜欢的发行版，例如 Ubuntu、Debian、Fedora 等。

如果您使用的是 Ubuntu，则可以从提供的二进制包进行安装：

sudo add-apt-repository ppa:srslte/releases
sudo apt-get update
sudo apt-get install srsepc

如果您使用其他发行版，可以参考项目的 GitHub 页面，按源代码方式安装。

安装完成后，您可以运行以下命令，将配置文件安装到默认位置（~/.config/srsran_4g）：

srsran_4g_install_configs.sh user

运行软件

要使用默认参数运行 srsEPC，请运行 sudo srsepc 在命令行上。 srsEPC 需要以 sudo 管理员权限运行才能创建 TUN 设备。这将启动 EPC 并等待 eNB 和 UE 连接到它。

srsEPC将启动TUN接口 srs_spgw_sgi 这将允许用户平面数据包到达 UE。

配置

EPC可以通过两个配置文件进行配置： epc.conf 和 user_db.csv。 的 epc.conf 将保存MME、SPGW和HSS的通用配置参数。 这包括 PLMN 值、完整性/加密算法、APN、SGi IP 地址、GTP-U 绑定地址等。

的 user_db.csv 用于保存 UE HSS 的特定参数。 这将包括 IMSI、身份验证算法、K、OP 或 OPc 等。

在以下小节中，我们将介绍 srsEPC 的一些常见配置案例：将新的 UE 添加到 HSS 数据库、在不同的计算机上运行 eNB 和 EPC，以及设置网络路由以使 UE 连接到互联网。

将 UE 添加到 HSS 数据库

添加 UE 时能够 user_db.csv HSS 将使用的数据库，您必须确保该文件中的参数与 UE 的 USIM 卡中存储的参数匹配。

特别相关的是 IMSI、身份验证算法、K 和 OP 或 OPc（如果使用 MILENAGE 算法）。 IMSI 是 SIM 卡的唯一标识符，K 是 HSS 和 UE 用于相互验证的密钥。

SIM 卡通常使用的身份验证算法是 MILENAGE，但也有测试 SIM 卡使用 XOR 身份验证。 如果使用MILENAGE算法，还必须知道使用的是OP还是OPc以及该参数对应的值。

一旦您知道这些参数，您就可以在 user_db.csv 中进行替换，其格式如下：

(ue_name),(algo),(imsi),(K),(OP/OPc_type),(OP/OPc_value),(AMF),(SQN),(QCI),(IP_alloc)

因此，如果您的 SIM 卡具有以下参数：

  • 里程算法

  • IMSI = 999700000000001

  • K = 00112233445566778899aabbccddeeff

  • 使用OPc

  • OPc = 63bfa50ee6523365ff14c1f45f88737d

您可以配置 user_db.csv 像这样：

ue1,mil,999700000000001,00112233445566778899aabbccddeeff,opc,63bfa50ee6523365ff14c1f45f88737d,9000,000000000000,9,dynamic

eNB 和 srsEPC 位于不同的机器上

默认情况下，srsEPC 配置为与 srsENB 在同一台计算机上运行。 当在单独的机器上运行带有 eNB 的 srsEPC 时，只需配置 mme_bind_addr 和 gtpu_bind_addr.

MME 绑定地址将指定 MME 将在何处监听 eNB S1AP 连接。 GTP-U 绑定地址应与 MME 绑定地址相同，除非您希望在与 S1AP 连接不同的子网上运行用户平面。

所以如果你想在有IP的接口上监听eNB 10.0.1.10，你可以这样做：

sudo srsepc --mme.mme_bind_addr 10.0.1.10 --spgw.gtpu_bind_addr 10.0.1.10

将UE连接到互联网

为了让UE能够连接到Internet，需要进行IP伪装。如果不进行伪装，Linux 内核将不会将数据包从一个子网转发到另一个子网。

要启用此功能，您可以运行一个便捷脚本 sudo srsepc_if_masq <out_interface>, 其中 输出接口 是将PC连接到Internet的接口。

警告

输出接口 不是 srs_spgw_sgi 接口，而是将 PC 连接到互联网的以太网或 WiFi 以太网。

观察结果

默认情况下，日志文件存储在 /tmp/epc.log。可以检查此文件以解决与 srsEPC 相关的任何问题。 日志文件可以有多个详细级别，可以在 epc.conf 或通过命令行。它们还可以在每层容量上启用，这在对特定层进行故障排除时非常有用。
