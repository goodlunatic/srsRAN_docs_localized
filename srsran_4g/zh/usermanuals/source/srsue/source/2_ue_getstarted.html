开始使用

要开始使用 srsUE，您需要一台带有基于 GNU/Linux 的操作系统和 SDR RF 前端的 PC。 SDR RF 前端是一种通用无线电设备，例如 Ettus Research USRP，它连接到您的 PC 并支持原始无线电信号的传输和接收。

如果您使用 Ubuntu 操作系统，则可以从提供的二进制包安装 srsUE：

sudo add-apt-repository ppa:srslte/releases
sudo apt-get update
sudo apt-get install srsue

如果您使用其他发行版，可以参考项目的 GitHub 页面，按源代码方式安装。

安装完成后，您可以运行以下命令，将配置文件安装到默认位置（~/.config/srsran_4g）：

srsran_4g_install_configs.sh user

运行软件

要使用默认参数运行 srsUE，请在命令行执行 sudo srsue。srsUE 需要以 sudo 权限运行，以便创建高优先级线程，并在成功接入网络后创建 TUN 设备。启动后，srsUE 会尝试查找您的 RF 前端设备并连接到本地小区。

如果 srsUE 成功连接到本地网络，它会创建一个 TUN 接口 tun_srsue。 TUN 接口可用作 PC 上的任何其他网络接口，支持进出网络的数据流量。

配置

UE 可以通过配置文件进行配置： ue.conf。该配置文件提供与工作频率、发射功率级别、USIM 属性、日志记录级别等相关的参数。要使用已安装的配置文件运行 srsUE，请使用 sudo srsue ~/.config/srsran/ue.conf.

配置文件中指定的所有参数也可以在命令行上覆盖。例如，要使用不同的 EARFCN 运行 UE，请使用sudo srsue ~/.config/srsran_4g/ue.conf --rf.dl_earfcn 3350.

srsUE 默认使用虚拟 USIM 卡，参数来自 ue.conf。这些参数是：

  • ALGO - 要使用的身份验证算法（MILENAGE 或 XOR）

  • IMSI - USIM 的唯一标识符

  • K - 与 EPC 中的 HSS 共享的密钥

  • OP 或 OPc - 操作员代码（仅与 MILENAGE 算法一起使用）

要成功连接到网络，这些参数需要与 EPC 的 HSS 中的参数匹配。 MILENAGE 是大多数网络使用的算法，XOR 则主要用于测试设备和测试 USIM 卡。OP 是运营商代码，OPc 是针对特定 USIM 计算得到的运营商代码，srsUE 两者都支持。

硬件设置

要使用 srsUE 通过无线方式连接到本地网络，您将需要 RF 前端和合适的天线。默认 EARFCN 为 3400（上行 2565MHz，下行 2685MHz）。为了减少 TX-RX 串扰，我们建议将 TX 和 RX 天线彼此成 90 度角。

srsUE 还可以通过有线连接使用。电缆配置和所需的 RF 组件将取决于您的 RF 前端。对于 RF 前端（例如 USRP），将 TX 连接到 RX 并确保至少 30dB 的衰减，以避免损坏您的设备。

有关有线连接的更多详细信息，请参阅 高级用法.

操作系统设置

srsUE 在具有标准 Linux 内核的用户空间中运行。为了获得最佳性能，我们建议禁用 CPU 频率缩放。要禁用频率缩放，请使用：

echo "performance" | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor

观察结果

要观察 srsUE 结果，请使用生成的日志文件和数据包捕获。

默认情况下，日志文件创建于 /tmp/ue.log。 srsUE 配置文件可用于指定网络堆栈每一层的日志级别并启用十六进制消息输出。支持的日志级别包括调试、信息、警告、错误和无。

日志消息采用以下格式：

Timestamp  [Layer ]  Level    Content

例如：

17:52:25.246 [RLC ]  Info    DRB1 Tx SDU

或者启用十六进制消息输出：

17:52:25.246 [RLC ]  Info    DRB1 Tx SDU
         0000: 8b 45 00 00 c7 f3 8b 40 00 01 11 d1 f6 c0 a8 03
         0010: 01 ef ff ff fa 92 55 07 6c 00 b3 ee 41 4d 2d 53

PHY 层日志消息具有其他详细信息：

Timestamp  [Layer]  Level  [Subframe] Channel:  Content

例如：

17:52:26.094 [PHY1]  Info  [05788]  PDSCH:    l_crb= 1, harq=0, snr=22.1 dB, CW0: tbs=55, mcs=22, rv=0, crc=OK, it=1, dec_time=  12 us

srsUE 应用程序支持两个级别的数据包捕获 - MAC 层和 NAS 层。 MAC 层捕获包括控制和数据流量，如果由网络配置，则将进行加密。 NAS 层捕获仅包括控制流量，不会加密。可以使用 Wireshark (www.wireshark.org) 查看数据包捕获 (pcap) 文件。
